## 用例 1：明日三餐推荐 + 购物清单生成

### 场景
用户每天晚上不知道明天吃什么，也不想动脑搭配营养。助理根据用户口味偏好、冰箱里现有的菜、以及营养目标，一键生成明日三餐计划 + 买菜清单。

### 用户触发
> "帮我安排一下明天的三餐，顺便看看需要买什么菜"

### AI 执行步骤
1. **读取库存** → 读取用户维护的「冰箱库存」文本/表格（本地 CSV 或简单的 Markdown 列表）
2. **识别约束** →
   - 用户的饮食偏好（如：少辣、不吃香菜、低碳）
   - 营养目标（如：减脂期、增肌期、控糖）
   - 时间约束（早餐要快、午餐带饭、晚餐在家做）
3. **生成菜单** → LLM 根据库存食材和约束生成三餐方案
4. **计算缺口** → 对比菜单所需食材和冰箱库存，生成购物清单
5. **输出结果** → Markdown 表格（三餐方案 + 购物清单）

### 所需工具
- 本地 CSV/文本文件：`fridge_inventory.csv`
- LLM（GPT-4 / Claude / 国产大模型）
- 可选：菜谱数据库 API（下厨房/豆果）

### 可执行命令（Playground 形态）
```bash
# 生成明日三餐计划
$ python scripts/meal_plan.py --date tomorrow \
    --inventory ~/Life/fridge_inventory.csv \
    --profile ~/Life/diet_profile.json \
    --output ~/Life/meal_plan_tomorrow.md

# 输出示例:
# 🍳 明日三餐计划（2026-04-17）
# 
# 早餐: 燕麦牛奶 + 水煮蛋 + 小番茄
#   ⏱️  准备时间: 5 分钟
#   💪 热量: 380 kcal | 蛋白质: 18g | 碳水: 45g
# 
# 午餐: 蒜蓉西兰花炒虾仁 + 杂粮饭（便当）
#   ⏱️  准备时间: 20 分钟
#   💪 热量: 520 kcal | 蛋白质: 32g | 碳水: 55g
# 
# 晚餐: 番茄蛋汤 + 清炒油麦菜 + 蒸南瓜
#   ⏱️  准备时间: 15 分钟
#   💪 热量: 310 kcal | 蛋白质: 14g | 碳水: 35g
# 
# ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
# 🛒 购物清单（需补充）
#   - 虾仁 200g
#   - 小番茄 250g
#   - 油麦菜 300g
#   （其余食材冰箱已有 ✅）
```

### 成功标准
- [x] 三餐方案中 70% 以上食材来自现有库存
- [x] 购物清单可以直接截图发给买菜 App 或跑腿
- [x] 总热量和营养配比符合用户设定的 diet_profile

### 失败处理
- **库存文件不存在**：提示"还没有记录冰箱库存，先帮你创建一个模板"
- **库存食材快过期**：优先推荐使用即将过期的食材
- ** diet_profile 缺失**：使用默认均衡饮食配置，并提示用户补充偏好

## 用例 2：周末 Meal Prep 批量备餐计划

### 场景
上班族希望周末花 2 小时准备好一周的工作日便当，减少平时做饭时间。

### 用户触发
> "帮我做一个下周的 meal prep 计划"

### AI 执行步骤
1. **读取约束** → 用户人数、便当盒数量、冰箱库存
2. **设计菜谱** → 选择 3-4 道适合批量制作、冷藏后口感不差的菜
3. **排期优化** →
   - 哪些菜可以同时蒸煮（省燃气/电费）
   - 哪些需要先炒后烤
   - 分装顺序（先装不易串味的）
4. **生成执行手册** → 买菜清单 + 步骤清单（带时间轴）+ 分装指南
5. **输出结果** → Markdown 报告

### 可执行命令（Playground 形态）
```bash
$ python scripts/meal_prep.py --week 2026-W17 \
    --meals 5 \
    --inventory ~/Life/fridge_inventory.csv \

# 输出示例:

# 选定菜谱:
#   A. 黑椒鸡排 + 蒜香西兰花

#   C. 孜然鸡胸肉 + 凉拌黄瓜

#   肉类: 鸡胸肉 800g, 鸡腿肉 600g, 牛腩 500g
#   蔬菜: 西兰花 1kg, 芦笋 600g, 黄瓜 4 根
#   调料: 黑椒酱 1 瓶（假设家中没有）

# ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

#   09:00 牛腩焯水、腌制鸡肉
#   09:20 番茄牛腩进高压锅（40分钟）
#   09:30 鸡排进烤箱（25分钟）
```

### 成功标准
- [x] 时间轴合理，总耗时控制在 2.5 小时以内
- [x] 菜谱选择考虑了「冷藏复热口感」
- [x] 买菜清单可以一键复制到买菜 App

### 失败处理
- **用户没有便当盒**：在报告中追加"建议购买 5 个 750ml 分格饭盒"
- **食材过敏冲突**：检测到过敏原时立即标红并替换菜谱

## 用例 3：通勤路线智能规划

### 场景
用户每天通勤前想知道今天走哪条路最快，结合实时交通、天气、以及日历中的第一个外出目的地。

### 用户触发
> "我现在去公司，哪条路最快？"

### AI 执行步骤
1. **获取起点** → 用户当前位置（Tauri 可调用系统位置，Web 可让用户输入）
2. **获取终点** →
   - 默认：用户预设的公司地址
   - 或读取今日日历第一个外出事件地点
3. **获取实时信息** → 天气、交通拥堵指数、公共交通班次
4. **生成方案** → 对比 2-3 种出行方式（地铁、公交、打车、骑行）
5. **输出结果** → 推荐最优方案 + 备选方案

### 所需工具
- 高德/百度地图 API（路线规划 + 实时路况）
- 天气 API（心知天气 / OpenWeatherMap）
- 日历 API（本地 ICS / 飞书日历）

### 可执行命令（Playground 形态）
```bash
$ python scripts/commute_plan.py \
    --from "当前位置" \
    --to "公司" \
    --mode all \
    --output terminal

# 输出示例:
# 🚗 通勤规划（现在 → 公司）
# 距离: 12.5 km | 预计到达: 09:12
# 
# ⭐ 推荐: 地铁 2 号线
#   步行 8 分钟 → 地铁 18 分钟 → 步行 5 分钟
#   总耗时: 31 分钟 | 费用: ¥4
#   备注: 当前 2 号线拥挤度中等，下一班 3 分钟后到站
# 
# 备选 1: 打车
#   预计 26 分钟 | 费用: ¥28
#   备注: 北三环轻微拥堵，建议走东四环
# 
# 备选 2: 骑行
#   预计 42 分钟 | 费用: ¥0
#   备注: 今天有轻度雾霾，AQI 128，不太建议
# 
# ⚠️ 提醒: 你 10:30 在望京有会议， commute 时间刚好，但建议 10:00 提前出发。
```

### 成功标准
- [x] 能在 3 秒内返回路线结果
- [x] 自动关联今日日历并给出时间建议
- [x] 天气因素被纳入推荐逻辑（暴雨/雾霾时调整推荐）

### 失败处理
- **地址无法解析**：提示"请检查公司地址是否正确设置"
- **API 限额用完**：降级为显示预设的静态路线 + 平均耗时
- **无网络连接**：提示"当前无网络，显示的是历史平均通勤时间"

## 用例 4：差旅一键打包清单

### 场景
用户要出差，根据目的地天气、行程天数、会议/休闲安排，自动生成打包清单，减少忘带东西的烦恼。

### 用户触发
> "下周去上海出差 3 天，帮我列个打包清单"

### AI 执行步骤
1. **获取行程信息** → 目的地、天数、出发/返回日期
2. **查询天气** → 目的地期间的天气预报（温度、降雨概率）
3. **读取用户偏好** → 性别、常忘物品历史、是否有运动/游泳计划
4. **生成分类清单** → 衣物、洗漱、电子设备、证件、药品、其他
5. **输出结果** → 带勾选框的 Markdown 清单

## 用例 5：CSV 账单导入与自动分类

### 场景
用户每个月从支付宝/微信/银行导出账单 CSV，想快速整理成结构化的消费记录，省去手动分类的麻烦。

### 用户触发
> "帮我整理一下这个月的支付宝账单"

### AI 执行步骤
1. **读取 CSV** → 解析支付宝/微信/银行的标准 CSV 格式
2. **数据清洗** → 统一时间格式、去除退款/转账干扰项
3. **自动分类** →
   - 基于规则：商家关键词匹配（如"星巴克"→餐饮，"滴滴"→交通）
   - 基于 LLM：对规则未覆盖的交易进行智能分类
4. **异常标记** → 大额支出、陌生商户、重复扣款
5. **输出结果** → 清洗后的 CSV + 分类统计 + 可视化图表

### 可执行命令（Playground 形态）
```bash
$ python scripts/finance_import.py \
    --input ~/Downloads/alipay_record_202604.csv \
    --source alipay \
    --output ~/Finance/2026-04_consumption.csv

# 输出示例:
# 📥 导入完成: 187 条交易记录
# 🧹 清洗后有效记录: 163 条（去除 24 条转账/退款）
# 
# 自动分类结果:
#   餐饮美食      ¥3,240  (24 笔)  ████████████████████
#   交通出行      ¥1,180  (31 笔)  ███████
#   购物消费      ¥2,560  (12 笔)  ████████████████
#   生活服务      ¥  680  ( 8 笔)  ████
#   娱乐休闲      ¥  420  ( 3 笔)  ██
#   医疗保健      ¥  150  ( 1 笔)  █
#   未分类        ¥   80  ( 2 笔)  ⚠️  请手动确认
# 
# ⚠️ 异常提醒:
#   - 有一笔 ¥2,000 的支出给"XX 健身房"，属于大额消费
#   - 发现 2 笔相同金额相同时间的"滴滴出行"，疑似重复扣款
# 
# 清洗后的文件已保存，可直接导入记账 App。
```

### 成功标准
- [x] 主流账单格式（支付宝、微信、招商银行）能自动识别
- [x] 自动分类准确率 ≥ 85%
- [x] 异常交易能被自动标记出来

### 失败处理
- **CSV 格式不识别**：提示"当前只支持支付宝/微信/招商银行格式，请上传正确的文件"
- **编码错误（GBK/UTF-8）**：自动尝试多种编码解析
- **分类不确定**：标记为"待确认"，不胡乱分类

## 用例 6：月度消费报告生成

### 场景
用户想了解这个月钱花哪儿了，有没有超支，和下个月可以怎么优化。

### 用户触发
> "帮我分析一下 4 月份的消费情况"

### AI 执行步骤
1. **读取数据** → 导入已分类的消费记录 CSV
2. **计算核心指标** → 总支出、日均支出、环比变化、预算完成率
3. **深度分析** →
   - 哪类消费超预算最多
   - 哪些是不必要支出（基于用户历史标记）
   - 消费习惯洞察（如"周末餐饮支出占 60%"）
4. **生成建议** → 下月优化建议（可量化的）
5. **可视化输出** → 饼图、趋势图、对比图

### 可执行命令（Playground 形态）

```bash
$ python scripts/finance_report.py \
```

- [x] 预算超支时能指出具体超在哪里
- [x] HTML 报告可以在浏览器中打开，图表交互友好

### 失败处理
- **数据缺失**：提示"4 月份还没有导入账单，请先运行账单导入"
- **预算文件缺失**：只展示消费结构，不展示预算对比，并提示用户设置预算
- **只有一个月数据**：无法进行环比，报告中只展示当月情况

## 用例 7：订阅服务追踪与到期提醒

### 场景
用户有很多 App/网站的自动续费订阅（视频、音乐、云盘、会员等），经常忘了自己开了什么，花了多少钱，哪些快到期了。

### 用户触发
> "帮我看看现在有哪些订阅在扣费"

### AI 执行步骤
1. **读取订阅清单** → 用户维护的 `subscriptions.json` 或从账单中自动识别
2. **计算统计** → 月固定支出、年固定支出、即将到期的服务
3. **自动识别** → 从消费记录中扫描 recurring payment（每月固定扣款）
4. **生成提醒日历** → 在到期前 7 天/3 天/1 天设置提醒
5. **输出结果** → 订阅清单 + 费用汇总 + 到期提醒表
