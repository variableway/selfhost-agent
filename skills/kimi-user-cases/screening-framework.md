# 个人助理 Use Case 筛选与优先级框架

> 分析日期：2026-04-16

---

## 为什么需要筛选？

Personal Assistant 覆盖「衣食住行 + 学习 + 金融 + 娱乐 + 健康」等全生活场景，但资源（开发时间、API 成本、用户注意力）是有限的。**从宏大叙事落地到具体可执行的技能包**，需要一套清晰的筛选标准。

---

## 筛选四象限

```
                    高用户价值
                         │
    ┌────────────────────┼────────────────────┐
    │   战略储备区       │   立刻动手区       │
    │   (长期规划)       │   (P0 优先)        │
    │                    │                    │
低实现成本 ──────────────┼────────────────────┤ 高实现成本
    │                    │                    │
    │   顺手做掉区       │   谨慎评估区       │
    │   (P1 填充)        │   (需要 ROI 论证)  │
    │                    │                    │
    └────────────────────┼────────────────────┘
                         │
                    低用户价值
```

### 立刻动手区（P0）
- **高频**：用户每周甚至每天都会用到
- **痛点明确**：不做就很烦，做了就有爽感
- **可执行**：能用明确的命令/脚本/工作流实现
- **数据可得**：不需要打通太多封闭系统

### 顺手做掉区（P1）
- 实现成本低，虽然使用频率不高
- 可以作为主技能的附属功能
- 起到「惊喜时刻」的作用

### 战略储备区（P2）
- 用户价值高，但实现复杂或依赖外部生态成熟
- 先保持关注，等条件成熟再做

### 谨慎评估区（Pass）
- 成本高、价值低、替代方案多
- 暂时搁置

## 具体评估维度（打分卡）

| 维度 | 权重 | 5分标准 | 1分标准 |
|------|------|---------|---------|
| **使用频率** | 25% | 每天都会触发 | 一年用不到一次 |
| **痛点强度** | 25% | 不做就很焦虑/浪费时间 | 可有可无 |
| **技术可行性** | 20% | 现有 API/工具链完善 | 需要自研硬件或攻克技术难关 |
| **数据安全敏感度** | 15% | 可以本地处理或脱敏 | 必须上传大量隐私数据到云端 |
| **与 Playground 契合度** | 15% | 天然适合做成可执行技能教程 | 纯信息查询，不需要动手实践 |

**得分 ≥ 3.5**：进入 P0/P1 候选池  
**得分 2.5-3.5**：进入 P2 储备池  
**得分 < 2.5**：Pass

## 首批落地建议

基于以上框架，我建议从以下三个切口切入：

### 切口 1：Photo 摄影（P0）
**理由**：
- 话题具体，边界清晰
- 视觉大模型能力已经成熟（GPT-4V / Claude 3 / Qwen-VL）
- 用户痛点强：拍完照不知道好不好、不知道怎么修
- 在 Playground 中非常适合做「上传图片 → 执行分析脚本 → 看终端输出」的闭环

**落地方向**：
1. 单张照片技术分析（构图/曝光/参数）
2. 外拍后批量筛选和打分
3. 根据 EXIF 生成拍摄总结报告

### 切口 2：衣食住行 - 「食」与「行」（P0）
**理由**：
- 高频：每天都要吃饭、出行
- 信息来源公开：菜谱网站、地图 API、天气 API
- 不需要太多个人隐私数据
- 容易出「爽点」：一键生成明日三餐计划 + 购物清单

**落地方向**：
1. 根据冰箱库存/口味偏好推荐明日菜谱
2. 通勤路线规划（结合实时交通 + 日历目的地）
3. 差旅一键打包清单（根据目的地天气 + 行程天数）

### 切口 3：金融 - 个人记账与消费洞察（P0）
**理由**：
- 痛点强：很多人想理财但懒得记账
- 数据来源可控：支付宝/微信/银行导出的 CSV
- 分析逻辑标准化：分类、统计、可视化
- 非常适合 Playground 的「本地文件处理 + 终端命令」模式

**落地方向**：
1. CSV 账单导入与自动分类
2. 月度消费报告生成（图表 + 洞察）
3. 订阅服务追踪与到期提醒

|------|------|
| 家居环境控制 | 依赖智能家居生态，设备兼容性复杂 |
| 人际关系维护 | 情感权重高，AI 建议容易显得机械化 |
| 医疗健康诊断 | 合规风险高，不能替代专业医生 |
| 娱乐内容推荐 | 已经有 Spotify/Netflix/豆瓣等成熟产品 |
