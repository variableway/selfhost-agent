# Skill 对小白有用吗？Prompt 生成教程的设计思路

> 分析日期：2026-04-16

---

## 核心问题：Skill 对小白用户有价值吗？

**直接回答：有，但前提是不要让小白一开始就觉得"我在学 Skill"。**

### 为什么小白会抵触"Skill"？

| 小白的内心 OS | 原因 |
|--------------|------|
| "Skill 听起来很抽象，像是程序员才能用的东西" | 命名和概念门槛 |
| "我只要能解决今天这个问题就行" | 只关注即时结果，不关心体系化 |
| "看了教程还要配置环境，太麻烦了" | 对"教程"的印象停留在传统视频课 |

### 为什么 Skill 其实对小白很有价值？

**Prompt 是止痛药，Skill 是健身卡。**

- **Prompt**：解决一个具体问题，用完就忘，下次遇到类似问题还要重新搜
- **Skill**：把一类问题的解法固化成"可复制的工作流"，越用越顺手

**举例说明**：

| 场景 | Prompt 思维 | Skill 思维 |
|------|------------|-----------|
| 不知道照片怎么修 | "帮我看看这张照片"（下次还得再找人问） | 学会系统化的照片分析框架，任何照片都能自己诊断 |
| 明天吃什么 | "帮我规划明天的三餐"（每次都要重新描述） | 建立"冰箱库存 → 营养目标 → 菜单生成"的标准化流程 |
| 会议纪要 | "帮我整理这段文字" | 掌握"原始稿 → 结构化 → Action Items"的固定模板 |

**关键洞察**：
> 小白用户不是不需要 Skill，而是需要以"**先给 Prompt 尝到甜头，再引导理解背后的 Skill 框架**"的方式接触 Skill。

## 面向小白的 Skill 产品策略："三明治模型"

```
┌─────────────────────────────────────┐
│  第一层：Instant Prompt（即时满足）   │
│  "复制这个 Prompt，10 秒出结果"      │
├─────────────────────────────────────┤
│  第二层：Template Toolkit（模板库）   │
│  "保存这个模板，下次改几个词就能用"   │
├─────────────────────────────────────┤
│  第三层：Systematic Skill（系统化）   │
│  "理解为什么这个 Prompt 有效，        │
│   学会自己改、自己造"                │
└─────────────────────────────────────┘
```

### 在 Innate Playground 中的映射

| 层次 | Playground 呈现形式 | 小白感知 |
|------|---------------------|---------|

| Template Toolkit | 可下载的 Markdown 模板 + 变量说明 | "这个模板我可以保存" |
| Systematic Skill | 完整的可执行技能课程（MDX + 终端命令） | "原来背后是这么一套方法" |

## Prompt 生成教程的设计思路

### 思路 1：Prompt 诊所（Before / After 对比）

**最适合的场景**：让用户一眼看出"好 Prompt"和"坏 Prompt"的区别。

#### 教程结构

每个对比案例都包含：
1. **患者症状**：用户描述自己的需求（小白的真实表达）
2. **庸医处方**：一个常见的、效果差的 Prompt（小白通常会这样写）
3. **专家处方**：一个结构化、效果好的 Prompt
4. **药效对比**：两个 Prompt 的输出结果并排放在一起
5. **药理分析**：解释为什么专家处方更有效（逐步传递 Skill 思维）

#### 示例：会议纪要场景

**患者症状**：
> "我有一段会议录音转的文字，想整理成纪要，但不知道怎么写 Prompt。"

**庸医处方（❌）**：
```
帮我整理一下这段会议纪要。
【粘贴 3000 字文字稿】
```

**专家处方（✅）**：
```
请将以下会议录音转写稿整理成一份专业的会议纪要。

## 整理要求
1. **会议基本信息**：提取会议主题、时间、参与人
2. **会议议程**：列出主要议题
3. **关键结论**：用 bullet points 列出达成的共识/决定
4. **待办事项**：用表格呈现，包含事项、负责人、截止时间
5. **待确认问题**：列出需要后续跟进的问题

## 格式要求
- 使用 Markdown 格式
- 删除口语化表达和重复内容

---
【在这里粘贴文字稿】
```

**药效对比**：

| 维度 | 庸医处方结果 | 专家处方结果 |
|------|-------------|-------------|
| 结构 | 一大段文字，没有分层 | 清晰的五大模块 |
| 待办事项 | 散落在段落中，找不到 | 表格呈现，一目了然 |
| 可执行性 | 还需要手动二次整理 | 可以直接转发给团队 |
| 遗漏率 | 容易漏掉关键结论 | 按检查清单逐项提取 |

**药理分析**：
> 庸医处方只给了 AI 一个模糊的目标（"整理"），AI 不知道"整理到什么程度"。
> 专家处方做了三件事：
> 1. **定义输出结构**：告诉 AI 最终文档应该包含哪几个模块
> 2. **明确格式要求**：Markdown、表格、删除口语化
> 3. **分隔输入内容**：用 `---` 把 Prompt 指令和原始材料分开，避免 AI 混淆

### 思路 2：AI 擂台赛（多模型横向对比）

**最适合的场景**：打破小白"哪个 AI 最好用"的选择困难症，同时展示同一个 Prompt 在不同模型上的表现差异。

#### 教程结构

**比赛规则**：
- 同一道 Prompt（或同一类任务）
- 交给 2-3 个不同的 AI 模型（如 GPT-4、Claude 3.5、Kimi、豆包）
- 从多个维度打分
- 给出"什么场景用什么 AI"的选择建议

#### 示例：摄影诊断 Prompt 擂台赛

**擂台上的选手**：
- 🥊 **选手 A**：GPT-4o（OpenAI）
- 🥊 **选手 B**：Claude 3.5 Sonnet（Anthropic）
- 🥊 **选手 C**：Kimi k1.5（Moonshot）

**比赛用题**：
> 上传同一张逆光人像照片，使用统一的摄影诊断 Prompt

**评分维度**：

| 评分维度 | 权重 | GPT-4o | Claude 3.5 | Kimi |
|---------|------|--------|-----------|------|
| 构图分析准确度 | 25% | 9/10 | 8/10 | 7/10 |
| EXIF 读取完整性 | 20% | 7/10 | 6/10 | 5/10 |
| 后期建议可操作性 | 25% | 8/10 | 9/10 | 8/10 |
| 中文表达自然度 | 20% | 8/10 | 9/10 | 9/10 |
| 对小白的友好度 | 10% | 7/10 | 8/10 | 9/10 |

**擂台赛结论**：
- **追求分析深度**：选 GPT-4o（构图和光影判断最准）
- **追求表达质量**：选 Claude 3.5（建议最详细，措辞最像真人教练）
- **追求免费/国内可用**：选 Kimi（中文表达优秀，日常完全够用）

**附加洞察**：
> 如果你要发给摄影小白看，Claude 3.5 的输出可能最容易被接受；
> 如果你是专业摄影师想抠细节，GPT-4o 更值得付费。

### 思路 3：Prompt 改造工坊（从 0 到 1 的渐进式教学）

**最适合的场景**：让小白不只是"抄 Prompt"，而是学会"改 Prompt"。

#### 教程结构：四阶修炼

**第一阶：抄写（Copy）**
> "先不要想太多，直接复制这个 Prompt，感受一下效果。"

**第二阶：填空（Fill in the blanks）**
> "把 Prompt 中的【】部分换成你自己的信息。"

**第三阶：拼接（Mix & Match）**
> "这里有 3 个模块：角色设定 + 任务描述 + 输出格式。
>  试着把'角色'从'资深 HR'换成'创业公司 CEO'，看看输出风格有什么变化。"

**第四阶：原创（Create）**
> "现在你已经知道好 Prompt 的骨架了，试着自己写一个'旅行攻略生成 Prompt'。"

#### 在 Playground 中的可执行设计

每个阶都对应一个 `RunButton`：

```bash
# 第一阶：运行标准版 Prompt
$ python scripts/prompt_run.py --template photo_diagnose --input my_photo.jpg

# 第二阶：运行填空版 Prompt
$ python scripts/prompt_run.py --template photo_diagnose --fill blanks.json

# 第三阶：对比两种角色设定
$ python scripts/prompt_compare.py --template photo_diagnose --role "专业摄影师" --alt_role "摄影小白"

# 第四阶：验证用户自己写的 Prompt
$ python scripts/prompt_lint.py --user_prompt "帮我看看这张照片"
```

## 小白友好型 Prompt 教程的 5 个设计原则

### 1. 每一页都有"复制按钮"
小白不会手写 Prompt，教程页面必须提供一键复制。复制成功要有视觉反馈（toast / 按钮变色）。

### 2. 结果必须"看得见摸得着"
不要只给 Prompt 文字，必须附带：
- 输入示例（照片 / 文字稿 / 食材清单）
- 输出截图（完整的 AI 回复预览）
- 使用场景说明（"你可以在什么时候用这个 Prompt"）

### 3. 从"坏"到"好"，而不是只展示"好"
人性喜欢看"错误示范"和"翻车现场"。对比式教学比单向灌输效果好 3 倍以上。

### 4. 降低"保存成本"
提供：
- 飞书文档模板链接
- Notion 模板
- Markdown 文件下载
- iOS 快捷指令（把 Prompt 做成 Siri 语音触发）

### 5. 游戏化反馈
- "Prompt 诊所" → 治愈了多少个"患者"
- "AI 擂台赛" → 你支持哪个选手
- "改造工坊" → 完成一阶点亮一个徽章

## 与 Innate Playground 的结合：一个课程包设计

```yaml
course: Prompt 设计与 AI 工具使用（小白版）
difficulty: beginner
duration: 30 分钟
modules:
  - 01-copy-paste-wins:
      title: "复制就能赢：6 个立即可用的 Prompt"
      format: "Instant Prompt 合集 + 一键复制按钮"
  
  - 02-prompt-clinic:
      title: "Prompt 诊所：坏 Prompt vs 好 Prompt"
      format: "5 个对比案例（会议纪要 / 简历优化 / 三餐规划 / 摄影诊断 / 周报生成）"
  
  - 03-ai-battle:
      title: "AI 擂台赛：同一个 Prompt，谁答得最好？"
      format: "3 个模型 × 3 个场景，附带评分雷达图"
  
  - 04-prompt-workshop:
      title: "Prompt 改造工坊：从抄到造"
      format: "四阶修炼 + 终端命令验证"
  
  - 05-skill-upgrade:
      title: "从 Prompt 到 Skill：建立你的工作流"
      format: "把常用的 3 个 Prompt 升级成个人模板库"
```

### 每个模块的可执行块

1. `copy_prompt(template_name)` — 复制模板到剪贴板
2. `compare_outputs(bad_prompt, good_prompt)` — 本地运行并对比两个 Prompt 的输出效果
3. `ai_battle(prompt, models)` — 调用多个 AI API，生成对比报告
4. `prompt_lint(user_prompt)` — 分析用户写的 Prompt 给出改进建议
5. `build_template_library(prompts)` — 生成本地 Markdown 模板库文件

## 一句话总结

> **Skill 对小白有用，但入口必须是 Prompt。先让小白用 10 秒钟复制一个 Prompt 尝到甜头，再用"对比教学"和"AI 擂台赛"的方式，让他们理解"为什么这个 Prompt 好"，最终自然沉淀为系统化的 Skill 能力。**
