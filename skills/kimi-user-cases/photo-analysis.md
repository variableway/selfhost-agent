# 用例分析：摄影技术分析与图片处理

> 来源：`tasks/analysis/user-use-case/2-photo.md`
> 分析日期：2026-04-16

---

## 1. 用例概述

**核心目标**：让用户通过发送图片给 AI 代理（小龙虾/OpenClaw），获得专业的摄影技术分析反馈，并自动化完成图片的保存、归档和后续处理。

**价值主张**：
- 即时获取专业摄影点评（构图、曝光、色彩、后期建议）
- 打通"拍摄 → 上传 → 分析 → 归档"的完整工作流
- 支持个人摄影技能的成长追踪

---

## 2. 用户旅程拆解

### Step 1: 图片上传
用户通过聊天界面发送图片消息：
- 直接粘贴/拖拽图片到飞书/本地聊天窗口
- 批量上传一组照片（如一次外拍的作品集）

### Step 2: 代理接收与暂存
AI 代理接收图片并暂存：
- 提取图片元数据（EXIF：相机型号、镜头、光圈、快门、ISO、焦距）
- 生成缩略图和备份

### Step 3: 实时分析
AI 代理调用视觉大模型进行多维度分析：
| 分析维度 | 具体内容 |
|---------|---------|
| 构图 | 三分法、引导线、对称性、前景背景层次 |
| 曝光 | 高光/阴影细节、动态范围、HDR 建议 |
| 色彩 | 色温、饱和度、色调分离、配色方案 |
| 技术参数 | 基于 EXIF 的器材使用评价、参数优化建议 |
| 后期建议 | 调色方向、裁剪建议、局部调整提示 |
| 风格定位 | 人像/风光/街头/微距等类型判断 |

### Step 4: 结果反馈
分析结果以结构化方式返回给用户：
- 文字报告（Markdown 格式）
- 标注图（在图片上绘制构图线、过曝区域等）
- 对比图（原图 vs 建议后期效果，可选）

### Step 5: 保存与归档
用户选择保存方式：
- **本地相册**: 按日期/主题自动整理到文件夹
- **云端图库**: 上传到阿里云 OSS、腾讯云 COS、飞书云文档
- **作品集**: 标记为精选，加入个人摄影作品集

## 3. 技术实现要点

### 3.1 图片处理 Pipeline
```
用户上传图片
    │
    ▼
EXIF 提取 ─────────────┐
    │                  │
    ▼                  ▼
元数据结构化      生成缩略图/水印
    │                  │
    └────────┬─────────┘
             ▼
    多模态 LLM 视觉分析
             │
    ┌────────┼────────┐
    ▼        ▼        ▼
构图分析  曝光分析  后期建议
    │        │        │
    └────────┬────────┘
             ▼
    结构化报告生成 (Markdown/JSON)
             │
    ┌────────┼────────┐
    ▼        ▼        ▼
本地保存  云端同步  作品集更新
```

### 3.2 核心技术栈
| 功能 | 推荐工具/技术 |
|------|--------------|
| EXIF 读取 | `exiftool`, `piexifjs`, `exifr` |
| 图片处理 | `sharp` (Node.js), `Pillow` (Python), `ImageMagick` |
| 视觉分析 | GPT-4V, Claude 3 Vision, 国产多模态大模型 |
| 标注生成 | OpenCV / Pillow 绘制线条、热力图叠加 |
| 本地存储 | Tauri FS API (`@tauri-apps/plugin-fs`) |
| 云端同步 | 飞书 API / 对象存储 SDK |

### 3.3 EXIF 驱动的智能建议
基于相机参数自动生成技术反馈：
```javascript
// 示例逻辑
if (exif.iso > 3200 && exif.aperture > 'f/5.6') {
  advice.push("高 ISO 下建议开大光圈或使用三脚架降低感光度");
}
if (exif.shutterSpeed < '1/60s' && exif.focalLength > '50mm') {
  advice.push("长焦端快门速度偏低，可能存在手抖风险");
}
```

## 6. 关键挑战

1. **分析一致性**: 不同视觉模型对同一图片的评价可能差异很大，需要 prompt 工程标准化
2. **隐私保护**: 用户照片可能包含地理位置、人脸等敏感信息
3. **大图片处理**: 高像素 RAW 文件的读取和传输性能
4. **审美主观性**: 摄影评价具有较强的主观性，AI 建议应定位为参考而非标准
5. **批量分析成本**: 大量使用多模态 API 会产生较高费用，需要考虑本地模型替代方案
