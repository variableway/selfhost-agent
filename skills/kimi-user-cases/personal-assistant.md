# 用例分析：Personal 助理（个人全能助手）

> 来源：`tasks/analysis/user-use-case/3-personal.md`
> 分析日期：2026-04-16

---

## 1. 用例概述

**核心目标**：基于 OpenClaw 类工具为个人用户搭建一个覆盖「衣食住行、学习、金融、娱乐」等生活全场景的智能助理。

**价值主张**：
- 将重复性个人事务自动化（日程、提醒、信息整理）
- 提供跨领域的个性化建议和决策支持
- 成为用户的「第二大脑」，持续学习和适应用户偏好

---

## 2. 十大核心 Use Cases

### UC1: 日程与任务管理
**场景**：用户通过自然语言添加、修改、查询日程安排。

**示例**：
- "下周三下午 3 点提醒我去医院复诊"
- "帮我列出本周所有待办事项并按优先级排序"

**实现要点**：
- 对接日历 API（Google Calendar / 苹果日历 / 飞书日历）
- NLP 解析时间表达式（使用 `dateparser` / LLM 提取）
- 设置本地通知（Tauri notification API）

---

### UC2: 智能出行规划
**场景**：根据用户目的地、偏好、实时交通状况规划最优出行方案。

**示例**：
- "周五晚上从北京飞上海，帮我找最便宜的机票和附近的酒店"
- "明天早高峰去国贸，推荐最佳地铁换乘路线"

**实现要点**：
- 集成携程/去哪儿/高德/百度地图 API
- 多维度排序（价格、时间、舒适度）
- 自动添加到日历并设置出发提醒

---

### UC3: 饮食健康管理
**场景**：记录饮食、推荐食谱、分析营养成分。

**示例**：
- "今晚想吃点清淡的，推荐 3 个低卡菜谱"
- "帮我记录今天吃了什么，分析蛋白质和碳水摄入"

**实现要点**：
- 食物识别（图片 → 食材/菜品识别）
- 营养成分数据库对接（ USDA / 薄荷健康）
- 根据健康目标（减脂/增肌/控糖）生成饮食建议

---

### UC4: 个人财务管家
**场景**：记账、预算管理、消费分析、投资跟踪。

**示例**：
- "本月餐饮支出是多少？比上月多了多少？"
- "跟踪我持有的几只基金，每周发一次收益报告"

**实现要点**：
- 导入银行/支付宝/微信账单（CSV / API）
- 自动分类消费（LLM / 规则引擎）
- 可视化报表（echarts / recharts）
- 设置预算告警和储蓄目标

---

### UC5: 学习规划与知识管理
**场景**：制定学习计划、整理笔记、复习提醒、资料检索。

**示例**：
- "帮我制定 3 个月学会 Python 的学习路线图"
- "把我最近收藏的 20 篇文章总结成一个思维导图"

**实现要点**：
- 与 Readwise / Notion / Obsidian 对接
- 使用 Spaced Repetition 算法安排复习
- 知识库 RAG 检索（本地 embedding + vector DB）
- 与 Innate Playground 的技能学习系统天然融合

---

### UC6: 智能购物比价
**场景**：用户想买某件商品，助理自动全网比价、查优惠券、追踪降价。

**示例**：
- "想买索尼 A7M4，帮我找找现在最便宜的渠道"
- "AirPods Pro 降价到 1500 以下通知我"

**实现要点**：
- 电商爬虫 / 开放平台 API（京东、淘宝、拼多多）
- 价格历史追踪和降价预测
- 优惠券/返利信息聚合

---

### UC7: 家居环境控制
**场景**：连接智能家居设备，实现语音/文字控制。

**示例**：
- "我快到家了，打开空调并调到 26 度"
- "晚上 11 点自动关闭所有灯并启动睡眠模式"

**实现要点**：
- 对接米家 / HomeKit / Matter 协议
- 本地局域网控制（避免云端延迟）
- 场景自动化规则引擎（IFTTT 风格）

---

### UC8: 娱乐内容推荐
**场景**：基于用户口味推荐电影、音乐、书籍、游戏。

**示例**：
- "推荐几部类似《星际穿越》的科幻电影"
- "根据我最近听的歌生成一个周末放松歌单"

**实现要点**：
- 对接豆瓣 / Spotify / Netflix / Steam API
- 用户画像构建（偏好标签、历史行为）
- 多源内容聚合和去重

---

### UC9: 健康与运动追踪
**场景**：整合可穿戴设备数据，提供运动建议和健康预警。

**示例**：
- "这周平均每天走了多少步？睡眠质量怎么样？"
- "根据我的体重目标，推荐本周的运动计划"

**实现要点**：
- 对接 Apple Health / Google Fit / 华为运动健康
- 数据可视化和趋势分析
- 异常指标预警（心率、血氧、睡眠）

---

### UC10: 人际关系维护
**场景**：管理社交关系，提醒重要日期，辅助沟通内容。

**示例**：
- "下周日是母亲节，帮我挑一束花并写一段祝福语"
- "提醒我每个月给朋友 A 发一次消息保持联系"

**实现要点**：
- 联系人标签和关系图谱
- 重要日期提醒（生日、纪念日）
- 礼物/祝福语推荐引擎

## 4. 与 Innate Playground 的关联

Personal 助理的每个 Use Case 都可以转化为 Innate Playground 中的**可执行技能**：

### 课程设计
```yaml
course: 构建你的个人 AI 助理
difficulty: beginner → advanced
modules:
  - 01-assistant-basics: 助理架构与 OpenClaw 入门
  - 02-schedule-agent: 日历与任务自动化
  - 03-finance-agent: 个人财务分析
  - 04-learning-agent: 知识管理与学习规划
  - 05-smart-home: 智能家居控制
  - 06-health-agent: 健康数据追踪与分析
```

### 可执行块示例
```typescript
// 每个 Use Case 对应一组可执行命令
1. schedule_add("下周三下午3点去医院复诊")
2. travel_plan("北京", "上海", "2026-04-20")
3. finance_report("2026-04")
4. recipe_recommend(low_calorie=true, cuisine="中餐")
5. home_control(device="ac", action="on", temp=26)
```

### 终端交互价值
在 Playground 的终端面板中：
- 用户可以实时看到 Agent 调用 API 的过程
- 失败时可以手动修正命令并重新执行
- 通过 `RunButton` 一键触发预设的个人助理工作流

## 5. 扩展建议

| 扩展方向 | 描述 |
|---------|------|
| 多 Agent 协作 | 不同领域 Agent 可以互相调用，形成复杂工作流 |
| 本地优先架构 | 敏感数据（财务、健康）优先本地处理，不上云 |
| 语音交互 | 集成 Whisper + TTS，实现全语音助理体验 |
| 自动化脚本市场 | 用户可分享和下载其他人的助理脚本/工作流 |
| 硬件联动 | 与智能眼镜、智能手表等可穿戴设备深度集成 |
| 情感陪伴 | 基于长期交互历史，提供更个性化的情感支持 |

---

## 6. 关键挑战

1. **数据安全与隐私**: 个人助理涉及大量敏感数据，必须支持本地优先和端到端加密
2. **API 稳定性**: 第三方 API 变更频繁，需要抽象适配层和降级策略
3. **意图歧义**: "帮我订一辆车"可能指出租车、二手车、模型车，需要上下文澄清
4. **长期记忆**: 如何在成本和效果之间平衡长期记忆的存储和检索
5. **权限管理**: 不同 Agent 需要不同的系统权限，需要最小权限原则和用户授权机制
