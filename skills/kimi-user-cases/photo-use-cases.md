## 用例 1：单张照片技术诊断

### 场景
用户刚拍完一张照片，不确定拍得如何，想让 AI 快速给出专业的技术点评和后期建议。

### 用户触发
> "帮我看看这张照片拍得怎么样？" + 上传图片

### AI 执行步骤
1. **接收图片** → 保存到临时目录
2. **提取 EXIF** → 读取相机型号、镜头、光圈、快门、ISO、焦距
3. **视觉分析** → 调用多模态大模型，传入图片 + EXIF 数据
4. **结构化输出** → 生成 Markdown 格式的诊断报告
5. **可选操作** → 询问用户是否保存到「待修图」文件夹

### 所需工具
- `exiftool` / `exifr`：提取 EXIF
- 视觉大模型 API（GPT-4V / Claude 3 / Qwen-VL）
- `sharp` / `Pillow`：生成缩略图（可选）

### 可执行命令（Playground 形态）
```bash
# Step 1: 分析照片
$ python scripts/photo_diagnose.py ~/Downloads/IMG_001.jpg

# 输出示例:
# 📷 器材: Sony A7M4 + 24-70mm f/2.8
# ⚙️  参数: f/2.8, 1/125s, ISO 800, 35mm
# ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
# 构图: 三分法使用得当，但地平线略有倾斜（约2°）
# 曝光: 整体偏暗，阴影部分细节丢失
# 对焦: 主体眼睛清晰，景深控制合理
# 建议: 
#   1. 后期提亮阴影 +15
#   2. 旋转校正地平线
#   3. ISO 800 有轻微噪点，建议降噪
```

### 成功标准
- [x] 能在 5 秒内提取出完整 EXIF
- [x] 分析报告中包含「构图 / 曝光 / 对焦 / 后期建议」四个维度
- [x] 对非摄影用户也能看懂（避免过多黑话）

### 失败处理
- **EXIF 丢失**：提示"这张照片没有 EXIF，可能是截图或经过压缩"
- **API 超时**：降级为本地基础 EXIF 展示，不等待视觉分析

## 用例 2：外拍批量筛选与评分

### 场景
用户一次外拍拍了几百张照片，想快速筛选出「值得修的」和「可以删的」，减少后期工作量。

### 用户触发
> "帮我筛选一下昨天外拍的照片，找出最好的 10 张"

### AI 执行步骤
1. **扫描目录** → 读取指定文件夹内所有 JPG/RAW 文件
2. **基础过滤** → 自动删除/标记：严重过曝、严重失焦、闭眼（人像时）、重复连拍
3. **分组聚类** → 将相似场景/连拍照片分组，每组只留最佳 1-2 张
4. **打分排序** → 综合技术质量（曝光、锐度）+ 美学评分进行排序
5. **输出结果** → 生成 HTML/Markdown 报告，带缩略图和分数

### 所需工具
- `exiftool`：批量读取 EXIF
- `imagemagick` / `sharp`：生成缩略图、计算直方图
- OpenCV / `laplacian` 算法：检测图像锐度
- 视觉大模型 API：美学评分（每组抽 1 张代表）

### 可执行命令（Playground 形态）
```bash
# 基础筛选（本地完成，不调用 API）
$ python scripts/photo_cull.py ~/Photos/2026-04-15-waipo \
    --min-sharpness 100 \
    --remove-duplicates \
    --output ~/Photos/2026-04-15-waipo/selected/

# 输出:
# 扫描完成: 287 张
# 删除: 34 张过曝 | 12 张失焦 | 45 张重复连拍
# 剩余: 196 张
# 
# AI 评分中（预计 2 分钟）...
# Top 10 已保存到 selected/best_10.html
```

### 成功标准
- [x] 100 张照片能在 1 分钟内完成基础过滤
- [x] 重复连拍检测准确率达到 90% 以上
- [x] 最终推荐的 Top 10 中用户认可率 ≥ 70%

### 失败处理
- **RAW 格式不支持**：跳过该文件并记录，继续处理 JPG
- **文件夹为空**：友好提示"该文件夹没有找到照片"
- **API 限额不足**：只评分前 30 张，其余按技术分排序

## 用例 3：月度拍摄总结报告

### 场景
用户想回顾自己这个月的拍摄情况，了解拍了什么、用了什么器材、技术水平有没有提升。

### 用户触发
> "给我生成本月的拍摄总结"

### AI 执行步骤
1. **扫描相册** → 读取本月所有照片的 EXIF 和创建时间
2. **数据聚合** →
   - 拍摄总张数、总时长跨度
   - 最常用相机/镜头组合
   - 最爱用的焦段、光圈分布
   - 拍摄题材分布（人像/风光/静物/街头）
3. **趋势对比** → 与上月/去年同期对比（如果有历史数据）
4. **可视化** → 生成图表（焦段饼图、光圈分布柱状图、拍摄时间热力图）
5. **文字总结** → 结合数据生成一段有趣的个人拍摄报告

### 所需工具
- `exiftool`：批量元数据提取
- `pandas`：数据处理
- `matplotlib` / `echarts`：图表生成
- LLM：生成个性化文字总结

### 可执行命令（Playground 形态）
```bash
# 生成月度报告
$ python scripts/photo_monthly_report.py \
    --input ~/Photos/ \
    --month 2026-04 \
    --output ~/Reports/photo_report_2026-04.html

# 输出示例:
# 📊 2026年4月拍摄报告
# 本月拍摄: 1,247 张
# 活跃天数: 18 天
# 主力器材: Sony A7M4 + 85mm f/1.4
# 最爱焦段: 85mm（占 43%）→ 你很可能在拍人像
# 平均光圈: f/2.2 → 虚化爱好者确认无疑
# 
# 相比上月:
#   + 夜景拍摄增加 120%（4月樱花夜拍季？）
#   - 风光拍摄减少 30%
# 
# 建议:
#   你的 35mm 镜头这个月一次都没用，
#   下个月可以尝试一下人文扫街。
```

### 成功标准
- [x] 报告生成本地 HTML，包含至少 3 种图表
- [x] 能从数据中提取出有洞察的观察（不只是罗列数字）
- [x] 支持按指定月份回溯生成

### 失败处理
- **没有本月照片**：提示"这个月没有找到照片，试试别的月份？"
- **图表库缺失**：降级为纯文字报告 + 表格数据

## 用例 4：修图前后对比与建议可视化

### 场景
用户想知道某张照片「按照 AI 建议修图后会变成什么样」，想看看对比效果再决定是否动手。

### 用户触发
> "能不能模拟一下这张片修完后的效果？"

### AI 执行步骤
1. **接收原图**
2. **诊断分析** → 找出需要调整的地方（曝光、色温、对比度、裁剪）
3. **自动修图** → 使用 Python 图像处理库自动应用建议调整
4. **生成对比图** → 左右拼接原图和效果图
5. **输出参数** → 同时给出 Lightroom / Photoshop 的具体调整数值

### 所需工具
- `Pillow` / `OpenCV`：图像处理
- `colour-science`：色温调整
- LLM：生成专业后期建议

### 可执行命令（Playground 形态）
```bash
$ python scripts/photo_auto_edit.py ~/Photos/IMG_001.jpg --preview

# 输出:
# 诊断: 欠曝 / 色温偏冷 / 地平线倾斜 3°
# 自动调整:
#   - 曝光 +0.8
#   - 色温 +450K
#   - 对比度 +12
#   - 旋转 -3°
#   - 阴影 +20
# 对比图已保存: ~/Photos/IMG_001_preview.jpg
# 
# Lightroom 参数:
#   Exposure: +0.80
#   Temp: +45
#   Contrast: +12
#   Shadows: +20
#   Rotation: -3.0°
```

### 成功标准
- [x] 自动调整后的图片在视觉上确实有改善
- [x] 生成的对比图可以直接发给朋友征求意见
- [x] 给出的 Lightroom 参数可以直接照着调

### 失败处理
- **图片分辨率过高**：先压缩到 2000px 长边再处理
- **自动调整效果不佳**：提示"这张照片的后期建议较复杂，建议手动精修"
