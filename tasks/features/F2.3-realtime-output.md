# F2.3 实时输出通信（IPC + Events）

## Description

实现 Tauri Rust 后端与前端 xterm.js 的实时数据通信，让命令输出能流式显示在终端中。

## Acceptance Criteria

- [ ] Rust 后端 spawn 命令后，stdout 实时通过事件推送到前端
- [ ] 前端监听 `shell:stdout` 事件，将数据写入 xterm.js
- [ ] 前端监听 `shell:stderr` 事件，以红色/高亮样式显示
- [ ] 前端监听 `shell:exit` 事件，标记命令完成状态
- [ ] 多个命令可并行运行（多 session），互不干扰
- [ ] 通信延迟低，用户能看到"打字机"效果的实时输出

## Technical Notes

### Rust 端（emit events）

```rust
// 逐行读取 stdout 并 emit
let reader = BufReader::new(child.stdout.take().unwrap());
for line in reader.lines() {
    let line = line?;
    app.emit("shell:stdout", ShellOutput {
        session_id: id.clone(),
        data: format!("{}\n", line),
    })?;
}
```

### 前端（listen events）

```typescript
import { listen } from '@tauri-apps/api/event';

// 连接到终端 session
async function connectTerminal(sessionId: string, terminal: Terminal) {
  await listen('shell:stdout', (event) => {
    const { session_id, data } = event.payload;
    if (session_id === sessionId) {
      terminal.write(data);
    }
  });

  await listen('shell:stderr', (event) => {
    const { session_id, data } = event.payload;
    if (session_id === sessionId) {
      terminal.write(`\x1b[31m${data}\x1b[0m`); // 红色
    }
  });

  await listen('shell:exit', (event) => {
    const { session_id, code } = event.payload;
    if (session_id === sessionId) {
      terminal.write(`\r\n\x1b[33mProcess exited with code ${code}\x1b[0m\r\n`);
    }
  });
}
```
