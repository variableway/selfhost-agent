# F2.5 命令安全控制

## Description

实现 Shell 命令安全控制机制，限制可执行的命令范围，防止恶意命令执行。

## Acceptance Criteria

- [ ] 实现命令白名单机制（只允许 Skill JSON 中定义的命令）
- [ ] 危险操作（sudo、管理员权限）弹出确认对话框
- [ ] `curl | sh` 模式的命令需要二次确认
- [ ] Tauri capabilities 正确配置，限制 shell 插件权限
- [ ] 命令日志记录（记录所有执行的命令和时间）

## Technical Notes

### 命令分级

```
Level 0 - 安全（直接执行）:
  which, where, echo, node -v, python --version, git --version

Level 1 - 正常安装（直接执行）:
  brew install, npm install, fnm install, uv python install

Level 2 - 需要确认（弹出确认框）:
  curl | sh, 环境变量修改, 写入 .zshrc/.bashrc

Level 3 - 禁止:
  rm -rf /, format, 危险系统命令
```

### Tauri Capabilities

```json
// src-tauri/capabilities/shell.json
{
  "identifier": "shell-execute",
  "windows": ["main"],
  "permissions": [
    {
      "identifier": "shell:allow-execute",
      "allow": [
        { "cmd": "brew", "args": true },
        { "cmd": "npm", "args": true },
        { "cmd": "node", "args": true },
        { "cmd": "python3", "args": true },
        { "cmd": "fnm", "args": true },
        { "cmd": "uv", "args": true },
        { "cmd": "git", "args": true },
        { "cmd": "which", "args": true },
        { "cmd": "echo", "args": true }
      ]
    }
  ]
}
```
