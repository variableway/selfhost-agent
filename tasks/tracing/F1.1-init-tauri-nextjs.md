# F1.1 初始化 Tauri v2 + Next.js 桌面应用

## Description

在 `innate-next-mono/apps/desktop/` 下创建 Tauri v2 + Next.js 应用，作为桌面应用的起点。

## Acceptance Criteria

- [ ] 在 `apps/desktop/` 下成功创建 Tauri v2 项目
- [ ] Next.js 15 (App Router) 作为前端
- [ ] `pnpm dev` 能启动开发模式，Tauri 窗口正常显示 Next.js 页面
- [ ] 集成到现有 monorepo（pnpm workspace 正确配置）
- [ ] Tauri shell 插件已安装和配置
- [ ] Rust 后端能响应基本的 `#[tauri::command]`

## Technical Notes

```bash
# 在 innate-next-mono/ 下创建
pnpm create tauri-app apps/desktop --template next
# 或手动初始化：
cd innate-next-mono/apps/desktop
pnpm init
pnpm add @tauri-apps/cli @tauri-apps/api
pnpm tauri init
pnpm add @tauri-apps/plugin-shell
```

### 关键配置

- `tauri.conf.json` 中设置 `devUrl` 指向 Next.js dev server
- `next.config.js` 中配置 `output: 'export'`（Tauri 需要静态文件）
- 在 `capabilities/` 中配置 shell 插件权限

- 无（这是第一个任务）
