# Local AI Playground - 需求分析文档 (Gemini 3.1 版)

## 1. 产品背景与定位
**Local AI Playground** 旨在成为一款面向个人用户的轻量级本地 AI 环境搭建与使用工作台。
它的核心价值是“消除技术黑盒，降低 AI 使用与开发门槛”，通过可视化的 UI 引导和透明的终端执行过程，帮助技术背景薄弱的用户轻松搭建、使用甚至自己构建 AI 日常效率工具。

## 2. 目标用户画像
产品主要服务于两类人群：
1.  **零基础小白**：几乎没有编程经验，但希望能够使用最新的 AI 技术提升效率。他们需要“傻瓜式”的一键安装和现成的效率工具（如 AI 助手、翻译、图片生成等）。
2.  **有基础的初学者**：具备一定的学习能力和逻辑基础（如非计算机专业的大学本科毕业生）。他们不满足于单纯“使用”，更希望通过教程学习如何搭建环境、调用 API，甚至借助 AI 编写并运行简单的程序。

## 3. 用户核心旅程与使用场景 (Use Cases)
根据用户诉求，产品需要支持以下进阶式的用户旅程：
*   **Level 1 (环境配置)**：通过可视化的 UI 引导，完成基本内容（如依赖、环境变量）的一键安装与配置。
*   **Level 2 (API 启蒙)**：按照内置的交互式教程，一步步搭建自己本地使用的 AI API 调用环境。
*   **Level 3 (AI 编程初探)**：按照教程，使用 AI 辅助编写一些简单的程序并成功运行。
*   **Level 4 (开箱即用工具)**：直接使用系统内置的、别人写好的日常效率工具。
*   **Level 5 (高阶自建工具)**：利用系统提供的 UI Layout 和 ChatDashboard 等组件，拼装构建专属的日常效率工具（甚至结合 CLI 操作浏览器完成自动化任务）。

## 4. 产品核心能力清单
为了支撑上述用户旅程，工具必须具备以下能力：
*   **一键式配置中心**：提供友好的 UI 界面，帮助用户一键安装必要组件并配置好环境。
*   **互动教程与路线图 (ROADMAP)**：提供体系化的教程说明，支持边看边操作。
*   **开箱即用效率工具集**：内置 AI 助手、AI 翻译、AI 图片生成等基础工具。
*   **浏览器自动化引擎 (CLI to Browser)**：支持通过 CLI 操作浏览器完成较复杂的自动化效率任务。
*   **终端过程透明化**：在本地端执行操作时，必须一步步处理，同时将底层 terminal（终端）的实时内容展示出来，消除用户对底层操作的恐惧感。

## 5. 短期切入点与 MVP 范围
MVP (最小可行性产品) 将聚焦于以下几个切入点：
1.  **以 OpenClaw 为标杆的云端门户**：提供 OpenClaw 等类似工具的一键安装说明，并作为各类云 Claw 的聚合门户。
2.  **本地开发工具套件说明书**：集成代码相关工具、IDE（如 VS Code/Cursor）的安装教程与使用指南。
3.  **UI 组件库的“可视化游乐场”**：
    *   展示基础的 UI Layout 和组件使用说明，告诉用户“用这些组件结合 AI 能做出什么东西”。
    *   展示 ChatDashboard 等高阶组件的形态，让用户直观感受到“这能做什么”。

## 6. 产品表现形式
产品将分为两种端形态提供服务：
1.  **Web 端 (展示/引导态)**：提供产品展示、文档阅读、组件预览，但受限于沙盒环境，无法直接操作用户的本地机器。
2.  **本地端 (全功能操作态)**：可以直接操作本地机器（执行脚本、读写文件），并在 UI 中实时同步展示 Terminal 输出内容，是产品的核心形态。
