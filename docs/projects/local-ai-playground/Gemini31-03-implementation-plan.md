# Local AI Playground - 实施计划与路线图 (Gemini 3.1 版)

## 1. 整体实施策略 (Agile & MVP Driven)
本计划旨在将 **Local AI Playground** 从一个轻量级构想快速落地。我们采用敏捷迭代策略，优先完成 MVP（最小可行性产品），并在真实用户（非技术人员）反馈中不断迭代完善功能。

## 2. 第一阶段：核心骨架搭建与 OpenClaw 标杆 (Phase 1: MVP)

### 阶段目标：
搭建起客户端的基础设施，跑通“UI 配置 -> 脚本执行 -> 终端实时反馈”的核心体验闭环，并以 OpenClaw 的一键安装作为标杆。

### 关键任务与里程碑：
1.  **基础设施初始化**：
    *   技术栈确立与项目初始化（推荐 Electron/Tauri + React）。
    *   构建基础界面布局（左侧导航/说明，右侧/底部终端展示区）。
    *   引入 Xterm.js 并跑通本地 Shell 命令执行的测试。
2.  **配置向导模块 (Setup Wizard)**：
    *   设计 API Key 输入界面。
    *   编写并在后台调用基础安装脚本（如检测 Node.js、Python 等环境依赖）。
    *   **核心用例**：实现 OpenClaw 的一键安装与本地配置（如 `settings.json` 的自动写入）。
3.  **云端门户与基础展示**：
    *   提供各大云 Claw 服务的导航入口与介绍。
    *   提供基础的 IDE（如 VS Code、Cursor）安装教程及 AI 代码助手配置指南。

## 3. 第二阶段：教程体系与基础工具补齐 (Phase 2: Ecosystem)

### 阶段目标：
丰富内置工具和教程内容，降低小白用户的门槛，让他们不仅能“装好”，还能“用起来”。

### 关键任务与里程碑：
1.  **互动教程引擎 (Interactive Tutorials)**：
    *   开发基于 Markdown 的教程阅读器。
    *   实现“一键运行”功能，让教程中的代码块（如一段 API 测试脚本）可以直接点击执行，结果在内嵌终端即时呈现。
2.  **预置效率工具 (Built-in Productivity Tools)**：
    *   集成基础的 AI 助手模块（提供简易的 Chat 对话框）。
    *   集成基础的 AI 翻译、AI 图片生成等现成功能模块，方便小白直接调用。
3.  **Web 体验版发布**：
    *   剥离本地操作功能，发布只读模式的 Web 站点，用于概念展示、教程预览及引流。

## 4. 第三阶段：高阶玩法与自动化生态 (Phase 3: Advanced DIY)

### 阶段目标：
赋能有一定基础的用户，提供组件化能力与高阶浏览器自动化能力，让他们能够组合构建自己的日常效率工作流。

### 关键任务与里程碑：
1.  **可视化组件游乐场 (Component Showcase)**：
    *   提供 ChatDashboard、UI Layout 等核心组件的使用说明与效果预览（类似 Storybook）。
    *   展示不同组件结合 AI 能力后能构建出何种形态的应用。
2.  **浏览器 CLI 自动化 (Browser Automation)**：
    *   集成并封装轻量级的无头浏览器（如 Playwright/Puppeteer）。
    *   提供通过 CLI 或 UI 触发的高级日常自动化工具（如“一键打开网页并让 AI 总结核心内容”）。
3.  **用户自定义工具体系 (DIY Tools)**：
    *   允许用户通过编写简单的配置文件或组装系统提供的模块，生成个性化的本地效率工具。

## 5. 项目管理与文档规范

*   **文档先行原则**：在编写具体代码前，务必保证需求分析、架构设计和接口定义已清晰。
*   **轻量化验证**：每完成一个核心特性（如一键安装脚本），需经过目标受众（如不懂编程的普通用户）的真实体验反馈，确保“0 门槛”目标达成。
*   **代码规范与开源**：统一前后端技术栈代码风格，逐步向开源社区共建标准靠拢。
