# 即刻可见效果的 5 个场景

> 原则：一个动作，3 秒出结果，零配置，零学习。

前面的分析太重了。用户不会看完说明书再开始用。
他只会在打开 App 的那一刻决定"这东西有没有用"。

所以问题变成：**打开 App 后，用户做一个什么动作，能立刻看到"这不是聊天"？**

---

## 场景 1：📸 拍照秒出结果（最强 Wow 效果）

**用户动作**：拍照或选一张照片
**等待时间**：3 秒
**看到效果**：

```
用户拍了一杯咖啡的照片，屏幕上直接出现：

  ☕ 美式咖啡
  🔥 15 kcal（几乎为零）
  ☕ 咖啡因约 150mg

  比 ChatGPT 快，比搜索方便。
  因为不用打字。
```

**为什么这个最强**：
- 不需要打一个字
- 不需要想"我要问什么"
- 照片就是输入，结果就是输出
- 用户立刻理解：这不是聊天，这是**识物**

**实现成本**：只需要 GLM 多模态 API 一调用，一张卡片渲染。1 个接口 + 1 个组件。

---

## 场景 2：🎤 说一句话，屏幕上长出一张卡片

**用户动作**：对着手机说"午饭三十五"
**等待时间**：1 秒
**看到效果**：

```
屏幕上自动出现一张卡片：

  ┌─────────────────────────┐
  │ 🍜 午餐  ¥35            │
  │ 2026-04-16 12:30        │
  │                         │
  │ 今日已花: ¥35            │
  │ 本月餐饮: ¥1,280         │
  └─────────────────────────┘
```

**为什么有效**：
- 语音输入比打字快
- 卡片比文字直观
- 用户看到了"它不只是回复文字，它在帮我做事"
- 这张卡片就是**数据**，不是对话

**关键**：不是 Agent 回复一段文字"好的，已记录午餐35元"。而是**直接在屏幕上渲染一张结构化的卡片**。视觉上就区别于聊天。

---

## 场景 3：📊 打开就看到"你的数据"

**用户动作**：无。打开 App 就看到。
**等待时间**：0 秒
**看到效果**：

```
┌──────────────────────────────────┐
│                                  │
│  🌅 4月16日 周三                  │
│                                  │
│  ┌──────┐  ┌──────┐  ┌──────┐   │
│  │ 🍜   │  │ 💰   │  │ 📸   │   │
│  │ ¥35  │  │ ¥280 │  │ 3张   │   │
│  │ 今天  │  │ 本月  │  │ 今日  │   │
│  └──────┘  └──────┘  └──────┘   │
│                                  │
│  今日 1,850 kcal / 目标 1,800    │
│  ████████████████████░ 103%      │
│                                  │
│  [📸 拍一张]  [💰 记一笔]  [💬 聊] │
│                                  │
└──────────────────────────────────┘
```

**为什么有效**：
- 用户打开就看到**自己的数据**——不是空白对话框
- 数字就是最好的视觉冲击：¥280、1,850 kcal、3张照片
- 底部三个按钮：**拍照、记账、聊天**，对应三个最核心的动作
- 用户不需要想"我要做什么"，数据推着他走

**关键**：这是数据驱动的首页，不是功能列表页。用户看到的是**自己的消费、自己的饮食、自己的照片**，而不是"点击这里开始使用"。

---

## 场景 4：📥 粘贴一个链接，直接出摘要

**用户动作**：粘贴一个 URL
**等待时间**：5-8 秒
**看到效果**：

```
用户粘贴了: https://zhuanlan.zhihu.com/p/xxxxx

5秒后屏幕出现：

  📖 文章摘要

  标题: 如何用 AI 提高工作效率
  作者: 张三
  阅读时间: 约 8 分钟

  核心要点:
  1. 用 AI 写邮件，节省 60% 时间
  2. 用 AI 做会议纪要，不再手写
  3. 用 AI 整理笔记，自动分类

  [看全文] [保存到笔记] [生成 FlashCard]
```

**为什么有效**：
- 粘贴是最简单的操作（Ctrl+V）
- 用户有明确的期待（我想看这篇文章说了什么）
- 结果立刻可见（摘要 + 要点）
- 顺便展示了"不只是聊天"——它能**处理信息**

---

## 场景 5：⏰ 到点自动弹通知

**用户动作**：无。Agent 主动推送。
**等待时间**：0 秒
**看到效果**：

```
（手机/桌面弹出通知）

🦞 小龙虾提醒：
今天午饭花了 ¥35，比昨天少了 ¥15 👍
晚餐建议控制在 500 kcal 以内。
[拍照记录晚餐]

点击通知 → 直接打开 App → 摄像头已就绪
```

**为什么有效**：
- 用户什么都没做，小龙虾主动找他
- 这是**助手**和**聊天**最大的区别：助手主动，聊天被动
- 一条推送就建立了"这不是聊天工具"的认知

## 如果只能做一个，做哪个？

**📸 拍照秒识。**

理由：
1. **一个动作**：选图片或拍照，连字都不用打
2. **视觉冲击最强**：图片 → 文字 → 数字，三种信息形态在一张卡片里
3. **立刻证明"不是聊天"**：聊天要打字，拍照不用打字
4. **开发量最小**：GLM 多模态 API 一个调用 + 一个结果卡片组件
5. **可演示**：给别人看，对方"哦"的一声就懂了

---

## 最小实现：拍照秒识的 Demo

```
需要的代码：

1. 一个图片上传按钮（或拖拽区）
   → <input type="file" accept="image/*">

2. 一个 GLM 多模态调用
   → POST /api/analyze { image: base64, prompt: "识别并分析" }

3. 一个结果卡片组件
   → 识别名称 + 热量/描述 + 关键信息

就这三样东西。
不需要聊天框，不需要侧边栏，不需要路由。
一个页面，一个按钮，一张卡片。

用户打开 → 选照片 → 看到 → "哇"。
```
