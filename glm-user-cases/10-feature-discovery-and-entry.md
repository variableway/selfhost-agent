# 用户发现机制：如何让小白从"聊天"到"哇还能这样"

## 一、问题本质

用户打开 App，看到一个聊天框。他的心智模型是：

```
"这又是一个 ChatGPT 套壳吧？"
```

他不知道能拍照识热量、记账、生成食谱。**你不说，他永远不会点。**

核心矛盾：
- 聊天框是**万能入口**（什么都能做）→ 但对小白来说是**零暗示**（什么都不知道能做）
- 功能越多，发现成本越高

---

## 二、核心原则：不要"告诉"，要"触发"

| 错误做法 | 正确做法 |
|----------|----------|
| 新手引导弹窗（没人看） | 第一次打开时直接给一个"试一试" |
| 功能列表页面（像说明书） | 对话中的快捷按钮 |
| 设置页开关（藏得太深） | 聊天框旁的 "+" 按钮 |
| 文字说明"你可以拍照…" | 直接放一张示例照片让你点 |

> **关键洞察**：用户的"发现"不是发生在阅读说明的时候，而是发生在**顺手点了一下、立刻得到反馈**的时候。

## 三、三层入口设计

### 3.1 第一层：首页——"能力展厅"

用户打开 App 的第一个页面**不应该是对话框**，而是一个轻量的首页：

```
┌─────────────────────────────────────────────┐
│  🌅 早上好，Patrick                           │
│  今天想做什么？                               │
│                                              │
│  ┌───────────┐ ┌───────────┐ ┌───────────┐  │
│  │ 📸        │ │ 🍜        │ │ 💰        │  │
│  │ 拍照识热量 │ │ 生成食谱   │ │ 记一笔     │  │
│  │           │ │           │ │           │  │
│  │ 拍一张照片 │ │ 下周吃啥？ │ │ 午饭35    │  │
│  │ 就知道热量 │ │ 一键生成   │ │ 说就行    │  │
│  └───────────┘ └───────────┘ └───────────┘  │
│                                              │
│  ┌───────────┐ ┌───────────┐ ┌───────────┐  │
│  │ 📷        │ │ 📊        │ │ 📖        │  │
│  │ 拍照识物   │ │ 消费报告   │ │ 下载书籍   │  │
│  │ 万物识别   │ │ 这月花了多少│ │ 发链接就行  │  │
│  └───────────┘ └───────────┘ └───────────┘  │
│                                              │
│  ─── 最近 ──────────────────────────────────  │
│  💬 继续上次对话: "减脂食谱"                   │
│  📸 昨天: 红烧肉 560 kcal                     │
│  💰 本月已花: ¥2,340                          │
│                                              │
│  ─── 自由对话 ──────────────────────────────  │
│  [💬 随便聊聊...]                    [发送 →] │
└─────────────────────────────────────────────┘
```

**设计要点**：
- 6-8 个能力卡片，**每个都有具体的示例文案**（不是"拍照识别"，而是"拍一张照片就知道热量"）
- 用户点任何一个卡片 → 直接进入该功能的对话流
- 底部保留"自由对话"入口 → 兜底
- "最近"区域 → 回流钩子，让用户回到上次没做完的事

### 3.2 第二层：聊天框内的快捷入口

当用户进入聊天页面后，需要让他在对话过程中**随时能发现新能力**：

```
┌─────────────────────────────────────────────┐
│  💬 对话                                     │
│                                              │
│  Agent: 你好！我是小龙虾，有什么可以帮你？     │
│                                              │
│  ┌──────┐ ┌──────┐ ┌──────┐ ┌──────┐       │
│  │ 📸   │ │ 🍜   │ │ 💰   │ │ 📖   │       │
│  │ 拍照  │ │ 食谱  │ │ 记账  │ │ 读书  │       │
│  └──────┘ └──────┘ └──────┘ └──────┘       │
│                                              │
│  ┌───────────────────────────┐ ┌──┐ ┌──┐   │
│  │ 输入消息...                │ │📎│ │➤│   │
│  └───────────────────────────┘ └──┘ └──┘   │
└─────────────────────────────────────────────┘
```

**三个发现触发器**：

#### 触发器 1：对话建议气泡（Suggestion Chips）

首次对话后，Agent 主动展示可做的事：

```
Agent: 你好！我是小龙虾 🦞 不只是聊天，还能帮你：

  ┌────────────────┐ ┌────────────────┐
  │ 📸 拍照识热量   │ │ 🍜 生成食谱     │
  └────────────────┘ └────────────────┘
  ┌────────────────┐ ┌────────────────┐
  │ 💰 说句话就记账  │ │ 📷 拍照识万物   │
  └────────────────┘ └────────────────┘

  也可以直接打字告诉我你想做什么 👇
```

**关键**：这些不是静态按钮，是**带示例动作**的。点"拍照识热量"→ 直接弹出文件选择器/摄像头。

#### 触发器 2：输入框旁的 "+" 按钮

类似微信、飞书的附件按钮，但不是附件，是**能力快捷入口**：

```
点击 [+] 后弹出：

  📸 拍照          → 打开摄像头/选择图片
  💰 记一笔        → 快速记账表单
  🍜 生成食谱       → 进入食谱对话流
  📖 下载书籍       → 输入 URL
  📷 识别物品       → 打开摄像头
  📊 消费报告       → 直接显示本月报告
  ⚙️ 更多能力...    → 完整能力列表
```

#### 触发器 3：上下文感知提示

Agent 根据时间、用户历史**主动建议**：

```
（早上 8 点打开 App）
Agent: 早！今天想吃什么早餐？
       上次你说要减脂，要不要看看今天的减脂餐推荐？
       [看看推荐] [拍照记录早餐]

（月底打开 App）
Agent: 4月快结束了，要看看这个月的消费报告吗？
       [查看报告]

（用户刚发了一张照片）
Agent: 识别完成！要不要顺便记一笔？
       [记一笔 ¥35] [不用了]
```

### 3.3 第三层：对话内的"惊喜时刻"

用户在使用一个功能时，**意外发现**还能做另一件事：

```
用户: 拍了一张红烧肉照片
Agent: 📸 红烧肉，约 560 kcal

  顺便说一下：
  - 你这周已经吃了 3 次红烧肉了 😄
  - 如果想减脂，可以用清蒸鱼替换，热量只有一半
  - [记一笔] [看减脂食谱] [继续聊]
```

```
用户: 午饭35
Agent: ✅ 已记录 午餐 ¥35

  📊 顺便提醒：本月餐饮已 ¥1,280 / 预算 ¥2,000
  剩余 ¥720，日均可用 ¥48
  [看本月明细] [拍照记录明天的]
```

**核心逻辑**：每完成一次交互，**追加一个相关能力的入口**，但不强制。

## 四、首次使用流程（"Aha Moment"设计）

### 4.1 三步到达"Wow"

```
打开 App
  │
  ├── Step 1: 首页展示 6 个能力卡片
  │   用户看到 "📸 拍一张照片就知道热量"
  │   → 好奇心驱动，点了
  │
  ├── Step 2: 直接弹出图片选择（不是文字说明）
  │   用户选了一张食物照片（或用内置示例图）
  │   → 3秒后看到识别结果 + 热量
  │   → "Wow, 这也太方便了"
  │
  └── Step 3: Agent 追加："要不要记一笔？"
      用户点了 [记一笔]
      → 一步完成，零输入
      → "不用打字就能记账？？"
      → 用户被留住
```

**关键设计**：
- **不要求注册/配置**：第一次用不要求填任何信息
- **不要求输入 API Key**：内置几条示例图片，用 GLM 免费额度跑
- **不需要打字**：全程点击，3 秒出结果
- **内置示例**：如果用户没有食物照片，提供 3 张内置示例图

### 4.2 首页能力卡片的数据结构

```typescript
interface CapabilityCard {
  id: string;
  icon: string;                    // emoji
  title: string;                   // "拍照识热量"
  description: string;             // "拍一张照片就知道热量"
  entryType: 'photo' | 'text' | 'page' | 'action';
  entryAction: string;             // 点击后触发什么
  demoAvailable: boolean;          // 是否有内置示例
  demoData?: {
    imageUrl?: string;             // 示例图片
    sampleInput?: string;          // 示例输入文本
    sampleOutput?: string;         // 示例输出预览
  };
  order: number;                   // 显示顺序
  visible: boolean;                // 是否在首页显示
}
```

## 五、推荐入口方案对比

| 方案 | 优点 | 缺点 | 推荐度 |
|------|------|------|:------:|
| **A. 首页能力卡片** | 直观、零学习成本、每个能力一目了然 | 需要额外开发首页 | ⭐⭐⭐⭐⭐ |
| **B. 聊天框 "+" 按钮** | 不打断聊天流、符合微信习惯 | 需要点才能发现 | ⭐⭐⭐⭐ |
| **C. 对话建议气泡** | 跟随对话自然出现 | 只在首次/特定时机出现 | ⭐⭐⭐⭐ |
| **D. 侧边栏功能菜单** | 传统、清晰 | 太像后台管理系统 | ⭐⭐ |
| **E. 新手引导弹窗** | 什么都能塞 | 没人看完就关了 | ⭐ |
| **F. 命令斜杠 "/"** | 极客友好 | 小白不会用 | ⭐ |

**推荐组合**：**A + B + C**

```
首页能力卡片（初次发现）
  + 聊天框 "+" 按钮（随时发现）
  + 对话建议气泡（上下文发现）
  = 三层漏斗，确保用户 100% 触达
```

## 六、具体实现入口点

### 6.1 首页改造

当前 App 的路由：`/` → 教程首页

改造为：
```
/         → 能力首页（新）     ← 用户第一眼看到
/chat     → 对话页             ← 从首页或侧边栏进入
/tutorials → 教程首页（原 /）  ← 保留，但不再是默认页
```

### 6.2 聊天框改造

当前：纯文本输入框

改造：输入框左侧 + 按钮（附件/能力）

### 6.3 对话流改造

当前：Agent 回复纯文本

改造：Agent 回复可以包含**内联操作按钮**（Suggestion Chips）

## 七、数据驱动的发现优化

```typescript
// 追踪用户的能力使用情况
interface CapabilityUsage {
  userId: string;
  capabilities: {
    [key: string]: {
      firstUsedAt: number | null;   // 首次使用时间
      useCount: number;             // 使用次数
      lastUsedAt: number | null;    // 最近使用时间
    }
  }
}

// 优化逻辑：
// 1. 用户从未使用的功能 → 在建议气泡中优先推荐
// 2. 用户高频使用的功能 → 放在首页更靠前的位置
// 3. 用户连续 3 天使用某功能 → 在首页显示"常用"标签
```

**首页排序逻辑**：
```
固定前 2 位: 拍照识热量 + 记账（最核心）
动态排序  : 根据用户使用频率调整后面的卡片顺序
新功能    : 新上线的功能自动带 "NEW" 标签，展示 7 天
```

## 八、总结：从"聊天"到"生活助手"的转变路径

```
Day 1: 用户打开 → 看到能力卡片 → 好奇地点了"拍照识热量"
       → 拍了张午饭照片 → 3秒出结果 → "记一笔" → 惊喜 ✨

Day 3: Agent 早上主动问"今天早餐吃什么？"
       → 用户发现食谱功能 → "生成这周食谱"

Day 7: 月底，Agent 推送消费报告
       → "原来还能记账和看报告"

Day 14: 用户已经习惯每天拍照记录 + 语音记账
        → 开始探索更多功能（识物、读书...）

Day 30: 用户离不开小龙虾
        → "这比 ChatGPT 好用多了"
```

**核心公式**：
> **首次惊喜 × 低摩擦交互 × 持续主动触达 = 从聊天到生活助手**
