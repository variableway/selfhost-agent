# 优先级筛选分析：为什么从这些话题开始

## 一、筛选框架

选哪个 use case 先做，不是看哪个"听起来酷"，而是看哪个**做出来用户马上就能感受到价值**。我用了 5 个维度来打分（每项 1-5 分）：

| 维度 | 含义 | 权重 |
|------|------|------|
| **使用频率** | 用户多长时间会触发一次 | 30% |
| **即时价值** | 用户做完后立刻觉得"这东西有用" | 25% |
| **技术可达性** | 用当前 GLM API + 项目架构能否快速实现 | 25% |
| **数据依赖** | 是否需要外部 API/数据库，依赖越少越好 | 10% |
| **扩展潜力** | 做完后能否自然延伸到其他场景 | 10% |

> **核心判断逻辑**：频率高 × 即时价值大 × 技术能做 = 优先做。

## 三、为什么选这 3 个切入

### 切入点 1：📸 拍照识别（Photo 话题）

**为什么是它**：
- **具体到不能再具体**：拍一张照片 → 立刻出结果，零门槛
- **GLM 多模态天然适合**：视觉理解是大模型的核心能力，不需要额外训练
- **一个能力多个场景**：拍照识食物热量 → 拍照识植物 → 拍照识商品 → 拍照识文档
- **用户教育成本为零**：拍照谁都会

**衍生路径**：
```
拍照识热量 (MVP)
  → 拍照识物 (万物识别)
    → 拍照翻译 (OCR + 翻译)
      → 拍照做笔记 (文档 → Markdown)
```

### 切入点 2：🍜 饮食相关（食话题）

**为什么是它**：
- **每天 3 次的高频刚需**：没有人不需要吃饭
- **痛点明确**："今天吃什么"是世界级难题
- **闭环短**：输入偏好 → 输出食谱 → 生成购物清单，3 步搞定
- **拍照联动**：和 photo 话题天然结合（拍照 → 识别食物 → 计算热量）

### 切入点 3：💰 记账与消费分析（金融话题）

**为什么是它**：
- **自然语言记账是杀手级功能**：说一句话就记完一笔账，比打开 App 手动输入快 10 倍
- **数据完全自控**：不依赖外部 API，所有数据本地存储
- **积累效应**：记了 1 个月数据后，月度报告的价值指数级增长
- **技术简单**：GLM-4.5-air 提取结构化数据即可，不需要复杂推理

## 四、不优先做的场景（以及原因）

| 场景 | 暂缓原因 | 何时做 |
|------|----------|--------|
| 👔 穿搭推荐 | 需要衣橱数字化，前期投入大 | 有了拍照识物能力后自然延伸 |
| 📈 投资分析 | 金融数据 API 获取难，合规风险 | 积累用户信任后 |
| 🏠 买房分析 | 超低频，需要大量外部数据 | 平台成熟后 |
| 🎬 娱乐推荐 | 偏好学习需要大量交互数据 | 用户使用 1-2 个月后 |

## 五、落地顺序建议

```
Phase 0 (第 1 周)  ── 基础能力
  └─ GLM 多模态 API 调通（图片 + 文本）
  └─ 聊天消息类型扩展（支持图片附件）

Phase 1 (第 2-3 周) ── 📸 拍照识热量
  └─ 用户拍照 → GLM 识别食物 → 估算热量
  └─ 最小闭环：一张照片，一条分析

Phase 2 (第 3-4 周) ── 💰 自然语言记账
  └─ 用户说"午饭花了35" → 自动记录
  └─ 分类统计 + 简单月度报告

Phase 3 (第 5-6 周) ── 🍜 每周食谱 + 拍照联动
  └─ 根据偏好生成一周食谱
  └─ 拍照 → 识别 → 自动计入饮食记录

Phase 4 (第 7-8 周) ── 扩展
  └─ 拍照识物（万物识别）
  └─ 月度消费报告 + 预算分析
  └─ 食谱 + 购物清单
```

> **原则**：每个 Phase 都是一个独立的、用户可用的功能。不做半成品。
