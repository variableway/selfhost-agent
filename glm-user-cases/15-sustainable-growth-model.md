# 可持续性：每天一点点，可见可得

## 一、核心矛盾

OpenClaw 给了一个窗口：**低门槛，看到就能用**。

但问题来了：
- 免费 → 用户爽了 → 你亏了 → 倒了 → 用户走了
- 收费 → 用户还没看到价值 → 不会付费 → 没人用
- 太多推送 → 烦了 → 卸了
- 太少推送 → 忘了 → 不打开了

**核心问题不是"怎么收费"，而是"怎么让用户每天都愿意打开，每次打开都觉得值"。**

---

## 二、先回答：怎么持续保持"可见可得"

### 2.1 "每天一点点"的节奏设计

用户不需要每天花 1 小时。他只需要**每天 30 秒**。

但这 30 秒必须给他一个**具体的东西**，不是空洞的"今日小贴士"。

```
不是这样:
  ☀️ 今天也要加油哦！(× 没用)

而是这样:
  ☀️ 你昨天说"记得买牛奶"，买了没？
     [买了 ✅] [还没，提醒我] (✓ 具体)

  ☀️ 本月餐饮 ¥1,280，比上月同期少了 ¥200。
     [看明细] (✓ 数据驱动)

  ☀️ 你收藏的那篇"断舍离"，有 3 个待办还没做。
     [去做] (✓ 追踪闭环)
```

**原则：每次推送都是一条**需要用户做动作**的消息，不是纯信息。**

### 2.2 三种"每天一点点"

| 类型 | 示例 | 频率 | 目的 |
|------|------|:----:|------|
| **追问型** | "昨天说的牛奶买了没？" | 1-2次/天 | 闭环追踪，养成习惯 |
| **数据型** | "本月已花 ¥1,280" | 1次/天 | 积累效应，越用越有价值 |
| **发现型** | "有个 Prompt 模板你可能喜欢" | 1次/周 | 拓展认知，发现新能力 |

**关键比例**：追问型 > 数据型 >> 发现型。不要推太多"发现型"，会烦。

### 2.3 推送节奏（不超过每天 3 条）

```
08:00  追问型
       "今天早餐吃了什么？拍一张记录吧 📸"
       → 用户拍照 → 3秒识别 → 一天开始了

20:00  数据型
       "今日: 850 kcal · ¥35 · 2条待办"
       → 用户看一眼 → 心理满足感

22:00  (仅每周三) 发现型
       "这周最火的模板：30分钟减脂餐"
       → 用户点进去看看 → 可能会抄
```

## 三、收费不麻烦，关键在于"谁在什么时候付"

### 3.1 不要一开始就收费

```
第1-7天:    完全免费，随便用
            用户在积累数据（随手记、记账、拍照）

第8-30天:   免费额度 + 付费升级
            "本月已分析 50 张照片，升级 Pro 可无限"
            "月度报告已生成，Pro 可看完整版"

第30天+:    价值驱动付费
            用户已经有 30 天数据在这里了
            他不会轻易走
```

### 3.2 收费点设计

不是收"用 AI 的钱"，而是收**"帮你省的时间和帮你积累的数据"**。

| 收费点 | 怎么收 | 为什么合理 |
|--------|--------|-----------|
| AI 调用次数 | 免费额度 + Pro 无限 | 基础使用免费，重度用付费 |
| 高级模板 | 免费 5 个基础模板，Pro 全部 | 模板是内容，内容有价 |
| 月度/年度报告 | Pro 才有完整报告 | 数据分析是增值服务 |
| 多设备同步 | Pro 才有云同步 | 跨设备是硬需求 |
| 数据导出 | 免费可导出 CSV，Pro 可导出 PDF/可视化 | 导出格式是增值 |

**核心定价逻辑：基础功能永久免费，Pro = 更省时间 + 更多洞察。**

### 3.3 参考定价

```
免费版:
  - 每月 100 次 AI 分析
  - 5 个基础模板
  - 本地存储
  - 基础月度报告

Pro: ¥19/月 或 ¥168/年
  - 无限 AI 分析
  - 所有模板
  - 云同步 + 多设备
  - 完整月报 + 年度报告
  - 数据导出 PDF

家庭版: ¥39/月
  - Pro 全部功能
  - 最多 5 个家庭成员
  - 共享模板和食谱
```

**¥19/月** 的心理门槛非常低。一包烟的钱。

## 四、获取流量的三个渠道

### 4.1 社区推送（内容驱动）

```
渠道: 小红书 / 抖音 / B站 / 公众号

内容策略: 不是卖产品，是卖"抄作业"

小红书发帖示例:
  ┌────────────────────────────────────┐
  │ 📸 [截图: 小龙虾分析食物热量]        │
  │                                    │
  │ 标题: 发现一个神仙App！拍照就知道    │
  │       热量，不用打字不用搜 🔥        │
  │                                    │
  │ 正文:                               │
  │ 减脂姐妹看过来！这个小龙虾App...    │
  │                                    │
  │ ❤️ 2.3k  💬 156  ⭐ 1.8k          │
  └────────────────────────────────────┘
```

**关键**：每条内容都是一个**可直接使用的场景**，不是功能介绍。
- ✗ "小龙虾支持拍照识别"
- ✓ "拍一张午饭照片，3秒知道热量，我用了两周瘦了3斤"

### 4.2 模板传播（用户裂变）

```
用户 A 改了一个很好的减脂食谱模板
  → 分享给朋友 B（微信/链接）
  → B 打开链接 → 看到模板效果 → "我也要用"
  → B 注册 → 抄了 A 的模板
  → A 获得免费 Pro 天数奖励
```

**每个分享出去的模板都是一次免费广告。**

### 4.3 开源社区（技术信任）

```
项目本身开源（已经是了）
  → GitHub star 增长
  → 开发者关注
  → 技术博客/视频教程
  → 带来有基础用户
  → 有基础用户创造模板
  → 模板吸引小白用户
```

## 五、"可见可得"的持续状态怎么维持

### 5.1 用户生命周期设计

```
Day 1:    发现 → 拍照/发消息 → 立刻看到结果
Day 2-7:  每天被追问一次 → 养成习惯
Day 8-30: 积累数据 → 看到趋势 → 产生依赖
Day 30+:  模板推荐 → 发现新能力 → 深度使用
Day 90+:  分享自己的模板 → 成为贡献者
```

### 5.2 留存的关键指标

| 阶段 | 关键指标 | 目标 |
|------|----------|------|
| Day 1 | 完成至少 1 次拍照/记录 | > 70% |
| Day 3 | 累计 5 条记录 | > 50% |
| Day 7 | 回访（打开 App） | > 40% |
| Day 30 | 累计 50 条记录 | > 20% |
| Day 90 | 付费转化 | > 5% |

### 5.3 每个阶段推什么

| 阶段 | 推什么 | 不推什么 |
|------|--------|----------|
| 第1天 | 拍照识热量（最直觉） | 不要推功能列表 |
| 第1周 | 追问 + 数据统计 | 不要推模板 |
| 第2-4周 | 发现型内容（模板、新能力） | 不要推付费 |
| 第2月+ | 高级功能（月报、云同步） | 可以开始引导付费 |

## 六、总结：可持续公式

```
可持续 = 低门槛获取 × 每天30秒留存 × 数据积累 × 分享裂变

低门槛获取:
  社区内容（小红书/B站）
  + 模板分享链接
  + 开源社区

每天30秒留存:
  追问（"牛奶买了没？"）
  + 数据（"本月已花 ¥1,280"）
  + 不超过每天3条

数据积累:
  随手记 → 记账 → 拍照 → 心情
  → 数据越多 → 价值越大 → 越离不开

分享裂变:
  模板分享 → 新用户注册 → 又创造模板 → 又分享
```

**收费不是目标，是自然结果。当用户有 90 天数据在这里，¥19/月 他不会犹豫。**
