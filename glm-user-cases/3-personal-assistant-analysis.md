# GLM 用户用例分析：个人助理

> 来源：tasks/analysis/user-use-case/3-personal.md

## 一、用例概述

使用 OpenClaw（小龙虾）搭建个人 AI 助理，覆盖衣食住行、学习、金融、娱乐等日常生活场景。通过聊天交互实现任务自动化和智能建议。核心价值：**一个 Agent 覆盖生活全场景，越用越懂你**。

---

## 二、10 个核心 Use Case 详细分析

### UC-P1: 智能饮食规划助手

**场景描述**：
用户每天不知道吃什么，或者有饮食限制（减脂、健身、过敏），需要 AI 帮助规划三餐。

**交互流程**：
```
用户: 我这周想减脂，每天 1800 大卡，不吃辣
Agent: 🍽️ 为你生成本周减脂食谱（1800 kcal/天）

       周一:
       🌅 早餐: 燕麦酸奶碗 (380 kcal)
       🌞 午餐: 鸡胸肉沙拉 (520 kcal)
       🌙 晚餐: 清蒸鲈鱼+糙米饭 (600 kcal)
       🍎 加餐: 苹果+坚果 (300 kcal)

       [生成购物清单] [查看营养分析] [替换某餐]
```

**功能模块**：
- 食谱生成（GLM-5.1 + 营养知识）
- 营养计算（热量/蛋白质/碳水/脂肪）
- 购物清单生成（按食材聚合）
- 食材替换推荐
- 周计划管理 + 历史记录

**数据结构**：
```typescript
interface MealPlan {
  id: string;
  goal: '减脂' | '增肌' | '维持' | '控糖' | '自定义';
  dailyCalories: number;
  restrictions: string[];     // 忌口、过敏
  days: DayPlan[];
  shoppingList: ShoppingItem[];
}
```

**技术要点**：
- 营养数据库：可接入 USDA Food Data 或薄荷 API
- GLM 生成食谱时注入营养约束 prompt
- 支持图片识别（拍照识别食物热量）

### UC-P2: 穿搭推荐助手

**场景描述**：
根据天气、场合、个人衣橱，推荐每日穿搭。

**交互流程**：
```
用户: 明天去客户那开会，北京天气怎么样？
Agent: 📍 北京明天: 晴，12-22°C，微风

       👔 商务会议穿搭建议:
       - 深蓝西装 + 白衬衫 + 深色领带
       - 外搭风衣（早晚温差大）

       [拍照匹配衣橱] [保存搭配方案]
```

**功能要点**：
- 天气 API 集成（和风天气 / OpenWeatherMap）
- 衣橱数字化（拍照 → GLM 识别分类）
- 场合匹配（商务/休闲/运动/约会）
- 穿搭历史记录

### UC-P3: 智能出行规划

**场景描述**：
日常通勤优化、旅行规划、机票酒店比价。

**交互流程**：
```
用户: 下周末想去杭州玩两天，预算 3000
Agent: 🗺️ 杭州两日游规划（预算 ¥3,000）

       Day 1: 西湖 + 灵隐寺
       - 🚄 高铁: 周六 7:30 出发 ¥260
       - 🏨 酒店: 西湖附近民宿 ¥380/晚
       - 🍜 午餐: 知味观 ¥80
       - 🎫 灵隐寺门票 ¥75

       Day 2: 良渚 + 返程
       - ...

       预计总花费: ¥2,650
       [导出行程单] [预订提醒] [生成打卡清单]
```

**技术要点**：
- 高铁/机票查询 API（12306 / 携程开放平台）
- 地图 POI 推荐
- GLM 生成行程文本
- 预算计算器

根据用户目标制定学习计划，追踪进度，推荐资源。

**交互流程**：
```

       🎯 目标: 能用 Rust 写 CLI 工具和 Web 后端

       📖 推荐: The Rust Programming Language (免费)

       Month 2: 进阶实战
       - Week 5-6: 错误处理、泛型、Trait
       - Week 7-8: 异步编程、Tokio 入门

       Month 3: 项目实战
       - 开发一个 CLI 工具
       - 开发一个 REST API 服务

       [生成每日任务] [关联终端练习] [进度追踪]
```

**与项目集成**：
- 学习计划可导入到教程系统（Course → Tutorial）
- 练习步骤可在终端面板直接执行
- 进度追踪使用 Zustand store 持久化

### UC-P6: 个人财务管理

**场景描述**：
记账、预算管理、投资分析、财务报表。

**交互流程**：
```
用户: 记一笔，午饭花了 35
Agent: ✅ 已记录:
       🍜 午餐 | ¥35 | 2026-04-16

       📊 本月支出概况:
       餐饮: ¥1,280 (预算 ¥2,000)
       交通: ¥320  (预算 ¥500)
       娱乐: ¥200  (预算 ¥800)
       ────────────────
       合计: ¥1,800 / 预算 ¥5,000
       剩余可用: ¥3,200 (日均 ¥200)

       [查看分类详情] [导出月报] [设置预算]
```

**功能要点**：
- 自然语言记账（GLM 提取金额、分类、备注）
- 月度/年度报表
- 预算预警
- 投资收益追踪（基金/股票 API）

**技术要点**：
- 数据存储：tauri-plugin-store 或 SQLite
- 分类模型：GLM-4.5-air 提取消费类别
- 图表：recharts 月度趋势图

## 三、架构设计

### 3.1 整体架构

```
┌──────────────────────────────────────────────────┐
│                   聊天入口 (ChatPanel)             │
│              飞书 / 桌面 App / Web                  │
├──────────────────────────────────────────────────┤
│              意图识别路由 (GLM-4.5-air)             │
│   分析用户意图 → 路由到对应 Skill Handler            │
├──────────┬──────────┬──────────┬─────────────────┤
│  饮食     │  出行     │  学习     │  ... (10个Skill)  │
│  Handler │  Handler │  Handler │                  │
├──────────┴──────────┴──────────┴─────────────────┤
│                 共享服务层                          │
│  ┌─────────┐ ┌──────────┐ ┌─────────────────┐   │
│  │ GLM API │ │ 外部 API  │ │ 数据存储         │   │
│  │ Client  │ │ (天气/地图)│ │ (Store/SQLite)  │   │
│  └─────────┘ └──────────┘ └─────────────────┘   │
├──────────────────────────────────────────────────┤
│               记忆 & 偏好系统                      │
│   用户画像 · 历史偏好 · 上下文记忆                   │
└──────────────────────────────────────────────────┘
```

### 3.2 意图路由机制

```typescript
// 用户消息 → 意图识别 → Skill 路由
const intentPrompt = `
分析用户消息的意图类别，返回 JSON:
{
  "intent": "diet|clothing|travel|housing|learning|finance|entertainment|health|knowledge|schedule|general",
  "confidence": 0.0-1.0,
  "entities": { /* 提取的关键实体 */ }
}

用户消息: "${userMessage}"
`;

// 根据意图路由到对应 Handler
const handlers: Record<string, SkillHandler> = {
  diet: new DietHandler(),
  travel: new TravelHandler(),
  learning: new LearningHandler(),
  // ...
};
```

### 3.3 用户记忆系统

```typescript
interface UserProfile {
  // 基础信息
  dietRestrictions: string[];     // 忌口
  budgetLevel: 'low' | 'medium' | 'high';

  // 偏好 (从交互中学习)
  foodPreferences: string[];      // 喜欢的食物
  entertainmentPreferences: string[]; // 喜欢的电影类型
  learningGoals: string[];        // 学习目标

  // 上下文
  location: string;               // 所在城市
  recentTopics: string[];         // 最近话题

  // 关联数据
  expenseHistory: Expense[];      // 消费记录
  learningProgress: Progress[];   // 学习进度
}
```

## 四、实现路线图

### Phase 1: MVP (2-3 周)

选择 3 个最高频场景优先实现：

| 优先级 | Use Case | 原因 |
|--------|----------|------|
| P0 | UC-P1 饮食规划 | 高频、技术简单、效果直观 |
| P0 | UC-P6 财务管理 | 刚需、数据驱动、易量化 |
| P0 | UC-P10 日程管理 | 基础能力、与其他 UC 协同 |

### Phase 2: 扩展 (3-4 周)

| 优先级 | Use Case | 原因 |
|--------|----------|------|
| P1 | UC-P5 学习教练 | 与项目教程系统深度集成 |
| P1 | UC-P3 出行规划 | 需要 API 集成 |
| P1 | UC-P8 健康管理 | 日常高频场景 |

### Phase 3: 深度 (4-6 周)

| 优先级 | Use Case | 原因 |
|--------|----------|------|
| P2 | UC-P9 知识管理 | 需要搜索和图谱能力 |
| P2 | UC-P7 娱乐推荐 | 需要偏好学习 |

## 五、扩展方向

### 5.1 语音交互

接入语音识别（Whisper / 讯飞），实现语音助手模式：
- "小龙虾，今天午饭推荐什么？"
- "帮我记一笔，打车 28 块"

### 5.2 定时任务 / 主动推送

使用 OpenClaw 的 cron 功能：
- 每天 7:00 推送今日天气 + 穿搭建议
- 每周五生成本周消费报告
- 房租到期前 3 天提醒

### 5.3 多 Agent 协作

不同领域使用专业化 Agent：
```
用户消息
  → 路由 Agent (意图识别)
    → 饮食 Agent (食谱专家)
    → 财务 Agent (记账专家)
    → 学习 Agent (教学专家)
    → ...
  → 汇总 Agent (整合回复)
```

### 5.4 数据可视化仪表盘

个人数据总览页面：
- 本月消费环形图
- 运动频率折线图
- 学习进度甘特图
- 日程时间线
