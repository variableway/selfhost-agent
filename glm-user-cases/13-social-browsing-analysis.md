## 二、数据来源：怎么获取浏览记录

### 2.1 方案对比

| 方案 | 可行性 | 数据量 | 用户体验 | 推荐度 |
|------|:------:|:------:|:--------:|:------:|
| **A. 用户分享链接/截图** | 高 | 少量 | 主动操作 | ⭐⭐⭐⭐ |
| **B. 用户截图批量导入** | 高 | 中等 | 半自动 | ⭐⭐⭐⭐ |
| **C. App 导出数据** | 中 | 完整 | 需手动导出 | ⭐⭐⭐ |
| **D. 平台 API** | 低 | 完整 | 自动但难获取 | ⭐⭐ |
| **E. 浏览器插件** | 中 | 较完整 | 需装插件 | ⭐⭐⭐ |

### 2.2 推荐组合：A + B（最务实）

## 三、XHS 内容分析维度

### 3.1 单篇分析

| 维度 | 提取内容 | 用途 |
|------|----------|------|
| 标题/主题 | 这篇讲什么 | 分类标签 |
| 核心观点 | 3-5 个要点 | 知识提取 |
| 可操作项 | 步骤/清单/方法 | 生成 Todo |
| 质量评估 | 干货/营销/广告 | 过滤低质内容 |
| 情感倾向 | 正面/负面/中立 | 情绪分析 |

### 3.2 批量分析（日/周/月）

| 维度 | 分析内容 | 输出 |
|------|----------|------|
| 主题分布 | 这段时间关注什么 | 饼图 + 趋势 |
| 兴趣变化 | 和上周比有什么变化 | 趋势对比 |
| 收获总结 | 真正记住了什么 | 知识卡片 |
| 时间花费 | 在什么主题上花了多少时间 | 时间审计 |
| 情绪轨迹 | 浏览内容反映的情绪 | 情绪曲线 |
| 行动建议 | 哪些可以变成 Todo | Todo 列表 |

```
  "quality": "干货|一般|营销|广告",
  "sentiment": "positive|neutral|negative",
  "tags": ["标签1", "标签2", "标签3"]

```

### 4.2 批量分析 Prompt（日总结）

```
基于今天浏览/收藏的 ${count} 篇笔记，生成今日浏览报告。

语气：像朋友聊天，简洁，不说废话。
```

## 五、与"随心记录"的联动

这个功能和随手记天然联动：

```
随手记（万能入口）
  │
  ├─ 粘贴 XHS 链接    → 自动分析 → 收藏到笔记
  ├─ 截图批量导入      → 批量分析 → 提取 Todo
  │
  ├─ "今天看了什么"    → 触发浏览报告
  ├─ "这周收藏了什么"  → 周度整理
  │
  └─ 晚间自动推送
     "今天你看了 32 篇小红书，最有价值的是..."
```

**随手记的记录类型多一个**：

```typescript
type RecordType =
  | 'todo'
  | 'expense'
  | 'food'
  | 'mood'
  | 'bookmark'    // 已有
  | 'schedule'
  | 'moment'
  | 'note'
  | 'xhs-analysis';  // 新增：XHS 内容分析
```

## 六、这会是高频入口吗？

**是的，但频率取决于获取数据的方式。**

| 使用方式 | 频率 | 场景 |
|----------|:----:|------|
| 粘贴链接（单篇分析） | 3-10次/天 | 看到好内容就分享给小龙虾 |
| 截图批量导入 | 1次/天 | 晚上统一导入今天的截图 |
| "今天看了什么"（日总结） | 1次/天 | 睡前回顾 |
| 周度整理 | 1次/周 | 周末回顾这周收藏 |

**最高频的是"粘贴链接"——和随手记完全一样，就是发一条消息。**

## 七、更大的想象：不止小红书

```
XHS 分析 → 社交媒体浏览分析

同样的逻辑可以扩展到：
├── 小红书   → 生活/消费/美妆/旅行
├── 知乎     → 知识/观点/深度分析
├── 微博     → 热点/情绪/时事
├── B站      → 视频/教程/兴趣
├── Twitter/X → 技术/观点/国际资讯
├── Reddit   → 社区讨论/小众话题
└── 任意 URL → 通用内容分析

统一入口就是：粘贴链接 → AI 分析 → 分类保存
```

用户根本不需要告诉小龙虾"这是小红书的链接"。他粘贴任何链接，AI 自动识别来源、提取内容、分类保存。

**这才是一切回到"随心记录"的核心理念：用户只管发，AI 来分。**
