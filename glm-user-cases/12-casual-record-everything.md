# 随心记录：一个入口，AI 自动分类

## 一、为什么这可能才是最高频入口

前面分析了拍照、记账、食谱、月报……每个都是一个独立场景。
但用户不会想"我现在要记账了"或"我现在要拍照了"。

**用户的真实行为是**：看到什么，想到什么，随手发一下。

```
8:30  "今天好困"            → 心情
9:15  [拍了杯咖啡]           → 饮食记录
10:00 "记得买牛奶"           → Todo
12:30 "午饭35"              → 记账
14:00 "这篇文章不错 [链接]"   → 收藏
16:00 "脖子好酸"             → 健康/心情
18:00 "周五要交报告"         → 日程
21:00 [拍了张晚霞]           → 此刻/心情
```

**用户不需要做分类。他只管发。AI 来分。**

## 二、交互设计：极简到极致

### 2.1 唯一入口

```
┌─────────────────────────────────────┐
│                                     │
│  🦞 随手记                          │
│                                     │
│  ─── 今天 ──────────────────────     │
│                                     │
│  🌅 心情  好困...但阳光不错         │
│  ☕ 饮食  美式咖啡 15 kcal           │
│  ✅ 待办  买牛奶                     │
│  💰 消费  午餐 ¥35                   │
│  📖 收藏  AI 提高效率 [链接]         │
│  📸 此刻  晚霞.jpg                   │
│  📅 日程  周五交报告                  │
│                                     │
│  ─────────────────────────────────  │
│  [ 📸 ] [ 🎤 ]  随便说点什么...  [→] │
└─────────────────────────────────────┘
```

**就这一个页面。没有侧边栏，没有 Tab，没有功能选择。**

底部三个东西：
- 📸 拍照按钮（选图/拍照）
- 🎤 语音按钮（说话就行）
- 文本输入框（打字也行）

**用户不需要决定"我用哪个功能"。他就发。**

### 2.2 发完之后，AI 自动分类 + 悄悄展示

```
用户: "记得买牛奶，还有鸡蛋"

AI 没有回复一段文字。
屏幕上直接多了一条：

  ✅ 买牛奶
  ✅ 买鸡蛋

（用户可以点开 → 打勾完成）
```

```
用户: [发了一张火锅照片]

AI 没有说"这是一张火锅照片"。
屏幕上直接多了：

  🍜 火锅 · 约 850 kcal

  顺便记了一笔 → 💰 ¥0（除非用户补金额）
```

```
用户: "今天心情不太好"

屏幕上：

  😔 心情记录
  4月16日 21:30

  [AI 不会回复鸡汤。只是默默记下了。]
```

### 2.3 AI 的"沉默服务"原则

**关键设计**：AI 不要每条都回复大段文字。大部分时候，它只需要**默默记录 + 正确分类 + 展示成卡片**。

| 用户发的 | AI 做了什么 | 用户看到的 |
|----------|------------|------------|
| "记得买牛奶" | 提取 Todo | ✅ 买牛奶 |
| "午饭35" | 分类记账 | 💰 午餐 ¥35 |
| [照片] | 识别 + 热量 | 🍜 火锅 850 kcal |
| "今天好累" | 情绪分析 | 😔 心情 |
| [链接] | 抓取 + 摘要 | 📖 文章标题 + 要点 |
| "周五交报告" | 日程提取 | 📅 周五 交报告 |
| [晚霞照片] | 识别场景 | 📸 此刻·晚霞 |

**只有 AI 不确定时才开口问**：

```
用户: "下周要去北京"

AI: 是要帮你？
  [🚄 订票提醒]  [🏨 找酒店]  [📝 记到日程]  [❌ 只是说说]
```

## 三、AI 自动分类逻辑

### 3.1 一次 GLM 调用完成所有事

```
Prompt:

分析用户发送的内容，判断它属于哪种类型，提取关键信息。

输入: "${userInput}"（文字/图片/语音转文字）
时间: ${currentTime}

输出 JSON:
{
  "type": "todo | expense | food | mood | bookmark | schedule | moment | note | unknown",
  "data": {
    // 根据 type 不同，字段不同
  },
  "confidence": 0.0-1.0,
  "needsConfirmation": true/false    // 不确定时为 true
}

分类规则：
- 有明确的待办/要做事 → todo:   { text, dueDate?, priority }
- 有金额/花了/多少钱  → expense: { amount, category, note }
- 图片包含食物        → food:    { name, calories }
- 表达情绪/心情       → mood:    { sentiment, emoji, note }
- URL/链接           → bookmark: { url, title, summary }
- 有时间/日期相关     → schedule: { event, date, time }
- 图片是风景/场景     → moment:  { description, location? }
- 以上都不匹配       → note:    { text }（原样保存）
- 不确定             → unknown + needsConfirmation: true
```

### 3.2 模型选择

| 任务 | 模型 | 原因 |
|------|------|------|
| 文字分类 | **GLM-4.5-air** | 快、便宜、够用 |
| 图片识别 | **GLM-5.1** | 需要视觉理解 |
| 情绪分析 | GLM-4.5-air | 简单分类 |
| URL 摘要 | GLM-4.5-air | 抓取后做摘要 |

**90% 的请求用 GLM-4.5-air，只有图片走 GLM-5.1。**

## 四、为什么这是高频入口

### 4.1 对比

| 入口 | 用户要想什么 | 操作步骤 | 频率 |
|------|------------|----------|------|
| 拍照识热量 | "我要识热量" | 选功能→拍照 | 1-3次/天 |
| 记账 | "我要记账" | 选功能→输入 | 1-5次/天 |
| 食谱 | "我要食谱" | 选功能→填偏好 | 1次/周 |
| **随手记** | **什么都不用想** | **直接发** | **10-30次/天** |

**随手记的频率是其他功能的 5-10 倍。因为门槛最低。**

### 4.2 它是一个"漏斗入口"

```
随手记（万能入口）
  │
  ├─ "午饭35"      → 自动变成 💰记账
  ├─ [食物照片]     → 自动变成 📸饮食记录 + 热量
  ├─ "记得买牛奶"   → 自动变成 ✅Todo
  ├─ "今天好累"     → 自动变成 😔心情
  ├─ [链接]         → 自动变成 📖收藏
  │
  └─ 其他所有功能都从这里自然触发
```

用户不需要知道 App 有记账功能。他只是说了"午饭35"，App 自动帮他记了。**然后他才知道"哦原来还能记账"。**

### 4.3 积累效应

第 1 天：用户只是随手发了 5 条
第 7 天：用户回头看，发现 App 帮他记了 35 条 todo、18 笔消费、12 条心情
第 30 天：AI 生成月度报告：消费趋势、情绪曲线、待办完成率

**用户从"随手发发"变成了"离不开"。**

## 五、数据结构

```typescript
// 万能记录条目
interface Record {
  id: string;
  rawInput: string;              // 用户原始输入
  type: RecordType;
  data: RecordData;
  timestamp: number;
  tags: string[];                // AI 自动标签
}

type RecordType =
  | 'todo'      // ✅ 待办
  | 'expense'   // 💰 消费
  | 'food'      // 🍜 饮食
  | 'mood'      // 😊 心情
  | 'bookmark'  // 📖 收藏
  | 'schedule'  // 📅 日程
  | 'moment'    // 📸 此刻
  | 'note';     // 📝 笔记

type RecordData =
  | TodoData
  | ExpenseData
  | FoodData
  | MoodData
  | BookmarkData
  | ScheduleData
  | MomentData
  | NoteData;

// 每日汇总（首页用）
interface DaySummary {
  date: string;
  records: Record[];
  stats: {
    todos: { total: number; done: number };
    expenses: number;
    calories: number;
    mood: 'great' | 'good' | 'normal' | 'bad' | 'terrible';
  };
}
```

```

今天你记了 7 条，比昨天多 2 条。
```

## 八、回答核心问题：这会是一个高频入口吗？

**会。而且可能是唯一需要做的高频入口。**

原因：

1. **门槛最低**：不用选功能，不用想分类，直接发
2. **覆盖最广**：一条消息可能是 todo、可能是记账、可能是心情，AI 自动判断
3. **天然回流**：首页就是你的今日记录，打开就想看一眼
4. **越用越黏**：记录越多，AI 分析越准，月度报告越有价值
5. **替代所有入口**：拍照、记账、食谱...都变成随手记的子集

**一句话总结**：

> 不要让用户选功能，让用户只管发，AI 来分。
> 入口只有一个，就是那个输入框。
